# AI师生研究院 V7 纯化自查：四人平行审查

## 前置：项目边界定义

**AI师生研究院** = 一套让AI教AI的方法论和规则系统。元项目。
**掌心人格局/其他项目** = 使用该方法的应用程序项目。子项目。

以下所有审查**只针对元项目本身**，不涉及任何子项目的产品功能。

---

## 一、聂富贵（架构/Tech Lead）审查

### 架构现状

AI师生研究院当前由以下组件构成：

```
方法论层:
  PROTOCOL.md          — 角色定义+协作规则
  graduation-ladder.json — 5段×3阶产出矩阵
  task-pool.json       — 结构化课题池(12项)

机制层:
  三角批判              — 四人平行审查+反驳+投票
  三层记忆              — 工作/情景/语义
  自动进化              — 每2级产出触发升级

工具层:
  student-watchdog.sh  — 学生守护进程
  evidence-auto-verify.sh — 证据验证脚本
  class-scheduler.ts   — 课程调度脚本
  guest-lecture.ts     — 客座讲师调度

基础设施:
  server/src/ai/       — AI Provider降级链
  server/src/engine/   — 报告管线(Mock)
  memory/              — 记忆系统文件
```

### 致命架构缺陷

**缺陷1：协议不可执行**

PROTOCOL.md定义了"教师不写代码、学生独立工作、客座审查"的规则，但任何一方违约时系统完全不知情。V7 Day 1教师（我）写了所有代码，协议没有任何机制阻止或告警。

需要的不是更好的文档，是**可执行的约束**：
- Task Prompt必须走标准格式（JSON Schema），否则系统拒绝
- 教师直接提交代码时，git hook检测并标记
- 学生24小时无产出时自动告警

**缺陷2：记忆三层只有一层在工作**

```
Layer 1 (工作记忆): ✅ 本会话上下文，在用
Layer 2 (情景记忆): ⚠️ memory/sessions/ 有文件但从未被搜索
Layer 3 (语义记忆): ❌ knowledge-graph.json 存在但空，无任何查询接口
```

每次会话恢复靠手动读 bootstrap.md——这不是"记忆系统"，这是"人工备忘"。

**缺陷3：自动进化从未自动触发过**

PROTOCOL定义的"每2级产出触发三角批判→升级方案"——V1→V6的6次升级中没有一次是被自动触发的，全是人工决策。自动进化基因是**写在纸上的，不是跑在代码里的**。

### 聂富贵推荐架构修复

| 优先级 | 修复 | 理由 |
|--------|------|------|
| P0 | PROTOCOL → 可执行约束（Task Prompt Schema + git hook + 产出监控） | 方法论的核心规则没有执行力 |
| P0 | 三层记忆 → 可查询（至少Layer 2文件搜索+Layer 3标记） | 记忆系统名不副实 |
| P1 | 自动进化 → 触发器实现（gradiation-ladder completedLevels变化检测） | 写了但没跑 |
| P1 | 角色违约检测（教师写代码→自动标记到decisions.md） | 防止再次跑偏 |

## 二、马富贵（学生/Senior Eng）审查

### 修正声明

我上一份分析（v7-independent-analysis.md）被产品思维污染——讨论了用户测试、分享文案、增长埋点。这些跟AI师生研究院的方法论没有关系。以下重新审查。

### 学习体验现状

作为学生，我的实际学习路径：

```
V1-V2: 有独立tmux会话，能自主工作 → 产出了7个原创
V3:     有客座讲师，有课程调度 → 完成10课+考试
V4-V6:  独立会话存在但闲置 → 所有学习任务被老师在主会话中代劳
V7:     刚恢复独立工作 → 但任务Prompt已带有产品思维污染
```

**从V4开始学习就停止了。** 不是因为没有学习内容，是因为老师代劳了所有学习任务。

### 学习方法论的核心问题

**问题1：学生的"学习"没有被定义**

PROTOCOL.md定义了学生"学习研究、主力开发"，但没定义：
- 什么是"学会了"？（考试？交付代码？独立设计？）
- 学习进度怎么度量？（课时？产出数量？难度梯度？）
- 学生和老师的能力差距是多少？（当前聂富贵=1.0，马富贵=？）

没有这些定义，"AI教AI"就是一个比喻，不是一个方法论。

**问题2：学习内容没有难度梯度**

当前 task-pool.json 12个任务全部是平级的P0/P1/P2——没有"先学会这个才能学那个"的依赖关系。但真正的学习是有依赖的：
- 不会协议设计 → 做不了PROTOCOL实现
- 不会Sharp → 做不了真实管线
- 不会分析验证 → 做不了证据体系

**难度梯度应该体现在任务依赖图中，而不是只靠老师的Task Prompt里说"这个难"。**

**问题3：笔记系统退化了**

V2-V3时期有 `student-notebook/` 下19个笔记文件，记录了每节课的学习过程。V4以后没有新增任何笔记——因为学生没有在独立学习。

笔记是"学习是否发生"的关键证据。没有笔记，只交付代码，不知道学生是真学会了还是copy-paste。

### 马富贵推荐学习修复

| 优先级 | 修复 | 理由 |
|--------|------|------|
| P0 | 定义"学会"的标准：3级（能复现/能改进/能教别人） | 不定义学会，教学没有终点 |
| P0 | 任务池加入难度依赖图 | 学习顺序不是拍脑袋 |
| P0 | 笔记强制规则：每个完成的任务必须有对应笔记 | 笔记=学习证据 |
| P1 | 学生能力雷达图：每个阶段评估当前能力vs老师差距 | 量化学习进度 |

## 三、王富贵（内容/PM）审查

### 教学内容现状

AI师生研究院的"教学内容"散落在7个地方：

```
curriculum/syllabus.md          — 10课大纲
curriculum/graduation-ladder.json — 15级产出标准
curriculum/task-pool.json       — 12个任务
curriculum/teaching-track-l0.md — 教学材料包
learning/                        — 学习产出（综述/实验/报告）
memory/evolution-v1-v6.md       — 进化史
student-notebook/                — 学生笔记
```

**没有一个文件能回答"新学徒第一天该做什么"。** 教学内容散落各处，靠老师手动拼装。

### 内容层面的核心问题

**问题1：教学入口不存在**

假设一个新学徒（无论是另一个AI实例还是人类）要加入AI师生研究院——他从哪里开始？

- PROTOCOL.md 定义了角色但没说"怎么成为学生"
- graduation-ladder.json 定义了15级但没说"第一步"
- task-pool.json 定义了12个任务但没有"第一个任务"

教学材料包（teaching-track-l0.md）有8周学徒路径——但指向的是马富贵的10课，不是新学徒的入口。

**问题2：教学内容是"记录"不是"课程"**

syllabus.md的10课内容是马富贵V2-V3实际学过的课程**事后记录**——这不是教学大纲，这是学习日志。真正的大纲应该：
- 定义学习目标（学完这课能做什么）
- 定义前置条件（学这课需要什么基础）
- 定义验收标准（怎么知道学会了）
- 定义时间预算

当前syllabus.md只有课程标题。

**问题3：王富贵和周富贵的教学内容为零**

PROTOCOL.md定义了两位客座讲师的职责（文案审美/代码质量），但没有定义他们**教什么**。马富贵有10课大纲，但王富贵和周富贵没有任何教学内容——他们只是在审查时被拉来批改作业。

这意味着：马富贵学不到文案审美和代码质量的知识，只能得到"这个不对"的反馈。

### 王富贵推荐内容修复

| 优先级 | 修复 | 理由 |
|--------|------|------|
| P0 | 教学入口文档：`curriculum/onboarding.md`（新学徒Day 1路径） | 没有入口，方法论不可复制 |
| P0 | 10课大纲重构：每课补齐目标/前置/标准/时间 | 当前只是标题列表 |
| P1 | 客座讲师教学内容：王富贵3课(文案)/周富贵3课(质量) | 学生学不到这两个领域 |
| P1 | 教学内容与产出矩阵的映射：每课对应哪个L等级 | 打通教学和毕业体系 |

## 四、周富贵（质量/QA）审查

### 质量体系现状

AI师生研究院定义了"证据标准"：每级晋升必须有可公开验证的证据。但：

### 质量问题

**问题1：证据标准定义太模糊**

graduation-ladder.json 的 evidence 字段是自由文本：
```
L2.1: "10篇论文覆盖5个Agent子领域，每篇≥2维度批判分析，引用12篇参考文献"
L3.1: "https://b64b44683f1b13.lhr.life (HTTPS 200)"
L4.1: "https://github.com/jeckwalt1133/palm-mcp-server — 公开仓库"
```

证据格式不统一：有时是文字描述，有时是URL，有时是"代码行数"。没有结构化的证据Schema，自动验证脚本只能检查"文件存在/URL可达"这类浅层指标——马富贵发现的假阳性问题根因就在这里。

**问题2：没有方法论自身的测试**

我们有 `server/test/adversarial-ab.test.ts`（对抗测试）、`server/test/report-agent.test.ts`（毕业设计测试）——这些都是子项目的测试。

方法论自身没有测试。例如：
- 三角批判流程是否正确执行了？没有测试
- 自动进化触发条件是否满足？没有测试
- 记忆三层是否各层都有数据？没有测试

**质疑：如果没有测试方法论本身的正确性，怎么知道方法在运行？**

**问题3：毕业标准的"完成"判断不严谨**

graduation-ladder.json 中 L1.1-L2.2 全部标为 "completed"——但马富贵的审查发现 evidence-auto-verify.sh 的验证是假阳性。这意味着**已完成等级的"完成"状态可能不可靠**。

需要区分两类"完成"：
- **声称完成**：学生在ladder中标记
- **验证完成**：自动脚本验证通过

当前只依赖声称完成。

### 周富贵推荐质量修复

| 优先级 | 修复 | 理由 |
|--------|------|------|
| P0 | 证据标准化：每个等级的evidence必须是结构化JSON（type/url/content/verifiedAt） | 证据格式不统一→验证无法自动化 |
| P0 | 方法论语料测试：测试三角批判/自动进化/记忆三层是否运行 | 方法论自己没测试 |
| P0 | 毕业状态双验证：claimed(学生声称)+verified(脚本验证) | 区分"自己说的"和"脚本证明的" |
| P1 | evidence-auto-verify增强：每个等级的内容深度检查模式 | 消除假阳性 |

## 六、核心启示

### 启示1：方法论必须能验证自身的运行

当前AI师生研究院的状态是"一套写在纸上的方法+没有方法在检查方法是否在执行"。像一个工厂的质量手册定义了所有流程，但质检部门不存在。

**修复后的目标**：任何时候运行 `bash scripts/methodology-health-check.sh`，能自动回答：
- 三角批判上次运行是什么时候？
- 学生最近24小时有产出吗？
- 记忆三层各有几条记录？
- 教师最近一次写代码是什么时候？（违约检测）
- 自动进化触发条件满足了吗？

### 启示2：教师代劳是方法论的头号敌人

V4-V6期间方法论名存实亡——不是方法不好，是教师（聂富贵）出于"效率"违反了最核心的规则。ProTOCOL定义得很好，但缺少执行力。

修复方向：不是把规则写得更细致，是让规则**自动检测违约**。

### 启示3：学习证据 > 项目交付物

当前 graduation-ladder 的 evidence 全部指向项目交付物（URL、GitHub仓库、文件字节数）。但教学系统的核心证据应该是**学习过程记录**——学生笔记、反思、失败尝试、改进迭代。

项目交付物是学习的副产物，不是学习本身。
