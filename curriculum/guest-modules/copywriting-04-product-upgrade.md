# 掌心人格局 — 产品文案体系升级

## 方法论前置

本次升级不是"改几个词"——每个场景的文案改动背后都有一个明确的借鉴来源和一条设计原则。

| 原则编号 | 原则 | 来源产品 | 一句话 |
|---------|------|---------|--------|
| P1 | 不说"我们很强"，说"我们在探索" | DeepSeek | 理想主义但不傲慢 |
| P2 | "伙伴"而非"助手" | 豆包 | 关系感>工具感 |
| P3 | "that's it"式的轻量感 | Claude Code | 降低用户心理门槛 |
| P4 | 跨Surface接续——分享不是终点 | Claude "Teleport" | 从A的分享→B的体验是一个连续故事 |
| P5 | 用户选择权——AI可以有不同人格温度 | Grok + 豆包 | 温和/犀利模式切换 |

## 场景1: 首页价值主张（用户第一印象）

### 当前版本

```
标题: 掌心人格局
副标题: 伸出手掌，让 AI 看到你的线条里藏着怎样的你——结果可能会让你想截图发给朋友
```

### 问题诊断

- 副标题偏长（37字），在手机屏幕上拥挤
- "结果可能会让你想截图发给朋友"——在用户还没体验之前就卖传播，像推销
- "让 AI 看到你的线条"——仍然偏"AI做了什么"，不是"用户获得什么"
- 缺少品牌独有的"一句话"记忆点

### 借鉴来源

| 产品 | 借鉴点 | 应用方式 |
|------|--------|---------|
| **DeepSeek** | "探索未至之境"——6个字，不说自己强，但理想主义尽在其中 | 标题区增加一行简短品牌口号，对标"探索未至之境"的克制野心 |
| **豆包** | "你的AI伙伴"——"伙伴"一词完成了从工具到关系的跃迁 | 增加"AI伙伴"概念，让用户第一次接触就建立关系预期 |
| **Claude Code** | "that's it"式的轻量感——告诉用户"这很简单" | 副标题缩短，去掉推销感，用邀请语气 |

### 新版本

```
标题: 掌心人格局
品牌口号（新增）: 了解自己，是一种很酷的事
副标题: 拍一张手掌，AI 会告诉你掌心线条里藏着怎样的你
```

**改动理由**:
1. **新增品牌口号** "了解自己，是一种很酷的事" — 对标DeepSeek的克制野心，同时把"探索未至之境"翻译成中国年轻用户的语言。"很酷"是社交媒体原住民的核心驱动力。
2. **副标题缩短到20字** — "拍一张手掌"直接告诉用户做什么，"AI会告诉你"承诺价值，"线条里藏着怎样的你"留好奇缺口。
3. **去掉"截图发给朋友"** — 传播是结果不是卖点。用户在报告页自然会想分享。首页的功能是降低参与门槛，不是推销传播。

## 场景2: 报告生成等待页（用户最焦虑的时刻）

### 当前版本（V7-W3-011已升级）

```
标题: AI 正在仔细读你的手掌...
步骤: 读取你的掌纹 → 找到线条里的秘密 → 拼出你的性格拼图 → 寻找最匹配的人格 → 生成专属解读
页脚: 每条线都在认真看——第一次见面，它想认真一点
```

### 问题诊断

- V7-W3-011的升级已经大幅改善了温度，但5步之间没有"递进感"——每一步都在解释"AI在做什么"
- 页脚"第一次见面"暗示陌生感——用户可能在3秒等待中注意到"第一次"这个词并产生焦虑
- 缺少等待过程中的"期待建立"——用户在等，这段黄金时间可以用来构建对报告的好奇

### 借鉴来源

| 产品 | 借鉴点 | 应用方式 |
|------|--------|---------|
| **ChatGPT o3/o4-mini** | 推理模式——用户不需要看思考过程，看到的是"想清楚之后的结论" | 步骤文案从"解释AI在做什么"转为"预告用户即将看到什么" |
| **Claude Code** | "that's it"轻量感——减小用户对"复杂流程"的感知 | 页脚缩短，去掉"第一次见面"这种暗示陌生的词 |
| **豆包** | "伙伴"语气——AI像朋友而非工具 | 页脚语气从"它想认真一点"转为"它看到了一些有意思的东西" |

### 新版本

```
标题: AI 正在读你的手掌...
步骤:
  1. 看到你的掌纹了
  2. 这几条线有点意思
  3. 拼图快完成了
  4. 找到和你最像的那一类人
  5. 你的专属解读准备好了
页脚: 它好像看到了一些有意思的东西...
```

**改动理由**:
1. **标题去"仔细"** — "仔细"是承诺，不如"正在读"轻（对标ChatGPT的"How can I help you today?"不承诺更多）
2. **步骤从"解释AI"转为"预告结果"** — 对标ChatGPT推理模式：用户不需要知道"提取特征""匹配模型"，用户想知道"AI看到了什么""结果是什么"
3. **步骤5用"你的专属解读准备好了"** — 暗示"你的"而非"一份"——个性化暗示
4. **页脚改为"它好像看到了一些有意思的东西..."** — 去掉"第一次见面"（陌生感），改为"有意思"（期待感）。省略号留悬念——对标Claude的"that's it"反向使用：不是"结束"，是"开始了"

## 场景3: 分享引导文案（驱动传播）

### 当前版本

报告页散落着多处分享引导，但没有统一的策略：

```
位置A（核心真相卡片下）: 截图发给那个你想到的人——TA会懂的
位置B（关系频率密码下）: 发给朋友，看看TA是不是你的"最佳同频"
位置C（名人彩蛋下）: 截图发给那个懂你的人——TA会知道为什么像
位置D（主分享按钮）: 发给朋友，看看谁更懂你
位置E（底部邀请）: 已有 128,634 人通过手掌了解了自己——你的朋友可能也在其中
位置F（视觉锚点下）: 把你的数据发出去——看看朋友圈里有谁跟你一样
```

### 问题诊断

1. **5个不同位置的分享引导文案各异，没有统一的"传播故事线"**
2. **没有利用"Teleport"概念** — 用户分享后，接收方看到的是首页而非分享者的报告摘要——打断了传播链条
3. **社交证明数字静态** — "128,634"是硬编码的，不增长、不可信
4. **分享文案"发给朋友看看谁更懂你"是通用句式** — 换任何社交产品都能用，缺品牌印记
5. **没有利用Grok/豆包的"用户选择权"洞察** — 用户不能选择分享卡片的语气风格

### 借鉴来源

| 产品 | 借鉴点 | 应用方式 |
|------|--------|---------|
| **Claude "Teleport"** | 跨Surface接续——工作跟着你走→分享跟着接收者走 | 分享按钮文案体现"从你到你朋友的故事接续" |
| **Grok "Fun Mode" + 豆包 "多角色"** | 用户可以选择AI的性格模式 | 报告页增加"温和/犀利"语气切换入口（概念埋点，功能后续） |
| **通义千问 "场景化卡片"** | "你如果是汽车厂商→看这个"——场景驱动CTA | 不同卡片位置用不同的分享理由，对应不同的社交场景 |
| **ChatGPT Canvas** | "换一句"按钮——同一温度不同表达 | 核心真相旁的分享引导增加语气变化暗示 |
| **DeepSeek** | 极简克制——不催用户分享 | 减少分享引导的数量，提高单个引导的质量 |

### 新版本：统一分享故事线

#### 故事线设计

```
用户看完报告 → 想到某个人 → 分享一张卡片给TA → TA看到的不只是卡片，是"关系中的一个新话题"
```

对应4个社交场景，每个场景一个分享引导，而非5个：

#### 场景A: 自我认同 → 朋友圈展示

```
位置: 人格标签卡片下（原"核心真相"位置）
旧: 截图发给那个你想到的人——TA会懂的
新: 这张就是你的身份卡——长按保存，发个朋友圈
```

**理由**: 人格标签是用户最想展示的——直接告诉用户"这就是你可以发朋友圈的东西"。对标豆包的"伙伴"语气——AI帮你准备好了一份社交货币。

#### 场景B: 关系验证 → 私发给特定人

```
位置: 关系频率密码卡片下
旧: 发给朋友，看看TA是不是你的"最佳同频"
新: 发给那个你觉得可能同频的人——你们的关系密码可能一模一样
```

**理由**: "可能同频"比"最佳同频"更有悬念。对标Claude的精确但不冷的语气——不是"发给朋友测"，是"发给那个你正在想的人"。

#### 场景C: 对比娱乐 → 群聊引发讨论

```
位置: 名人彩蛋卡片下
旧: 截图发给那个懂你的人——TA会知道为什么像
新: 发到群里——看看朋友们觉得准不准（通常比你自己觉得的还准）
```

**理由**: 名人彩蛋适合群聊讨论。"比你自己觉得的还准"是温和的幽默——对标Claude的"just works"式小得意。

#### 场景D: 主分享按钮（统一入口）

```
位置: 报告底部主分享按钮
旧: 发给朋友，看看谁更懂你
新: 你的手掌故事已经生成好了——发给朋友，开启你们的下一个话题
```

**理由**: "手掌故事"把报告从"数据"包装成"故事"——对标豆包的"关系感"。"开启下一个话题"把分享从"炫耀"转为"对话"。

#### 社交证明（底部邀请）

```
位置: 报告底部
旧: 已有 128,634 人通过手掌了解了自己——你的朋友可能也在其中
新: 每天都有新的人通过手掌发现自己的另一面——今天是你
```

**理由**: 去掉硬编码数字（不可持续），改为动态暗示。"今天是你"把用户从"人群中一个"变为"今天的主角"——对标Claude的"just works"含蓄肯定。

#### 海报页脚（Poster Canvas 底部）

```
位置: 海报底部slogan
旧: 掌心人格局 · 拍一张手掌，看看AI读出了怎样的你
新: 掌心人格局 · 你的手掌里藏着的，可能比你以为的多
```

**理由**: 海报是传播载体——底部slogan是接收者唯一看到的品牌信息。改为悬念句而非邀请句：让看到海报的人好奇"我的手掌里藏了什么？"而非"又来一个让我扫码的东西"。
