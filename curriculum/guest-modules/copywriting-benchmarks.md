# 掌心人格局 — 文案写作标准

## 如何使用

每当你需要写一段用户可见的文案（首页/按钮/提示/报告/海报/错误提示），对照这5条标准自检。每条标准下面有"标杆示例"——做到这个程度就算好。

---

## 标准1: 有具体的锚点（不是泛泛而谈）

**好的文案用具体行为/场景/细节做锚点，用户能"对号入座"。**

### ✅ 标杆

| 文案 | 为什么好 |
|------|---------|
| "你看起来很好说话，但心里有张严格的评分表。" | "很好说话"+"评分表"——两个具体行为锚点。用户读完立刻知道"这说的是我"。 |
| "你在深夜做的决定和白天做的，像两个人的选择。" | 具体时间锚点(深夜vs白天)+具体行为(做决定)。可验证——用户可以回忆自己的深夜决定。 |
| "你的相册里最多的不是自拍，是那些舍不得删的截图。" | 行为锚点(相册/截图)+情感锚点(舍不得删)。L1打得极准。 |

### ❌ 反例

| 文案 | 问题 |
|------|------|
| "你是一个认真的人。" | 零锚点。能套给任何人。 |
| "你的情绪感知力很强。" | 零行为。用户不知道怎么验证。 |

---

## 标准2: 温度匹配场景（不是所有地方都该热）

**首页/拍照页/进度条用温3-5°（邀请/安抚），报告内容用热5-7°（共鸣/刺痛），错误提示用温3-4°（自嘲/拟人）。**

### ✅ 标杆

| 场景 | 文案 | 温度 | 为什么匹配 |
|------|------|------|-----------|
| 首页邀请 | "伸出手掌，让 AI 看到你的线条里藏着怎样的你" | 温4° | 邀请语气。不索要，给好奇心留缺口。 |
| 报告洞察 | "你真正的累不是事情多，而是总在照顾别人的感受，却很少有人反过来照顾你的情绪。" | 热6° | 有强L1锚点("照顾别人""没人照顾你")支撑，温度够热但不烫。 |
| 错误提示 | "网络有点慢，AI 还在等数据——稍后再试" | 温3° | 拟人化("AI 还在等")把负面体验转为品牌印象。不说"失败""异常"。 |

### ❌ 反例

| 场景 | 文案 | 温度 | 问题 |
|------|------|------|------|
| 首页标题 | "AI 读取你手掌的几何特征" | 冷1° | 首页是第一印象——温度太低，好奇心没建立 |
| 等待页 | "首次分析需要3-5秒，请耐心等待" | 冷1° | 等待时用户已有焦虑——"请耐心"是提醒等待 |
| 错误提示 | "分析失败" | 冷0° | 用户最差的体验时刻，文案没有任何温度 |

## 标准3: 用户视角（不是机器做了什么，是用户获得了什么）

**用户不关心"特征提取""匹配模型"。用户关心"AI 在看什么""AI 看完了告诉我什么"。**

### ✅ 标杆

| 机器视角 ❌ | 用户视角 ✅ |
|------------|-----------|
| "上传手掌照片" | "伸出手，让 AI 看看你" |
| "提取手掌特征" | "读取你的掌纹" |
| "分析线条走向" | "找到线条里的秘密" |
| "匹配人格模型" | "寻找最匹配的人格" |
| "生成专属报告" | "生成专属解读" |

### 翻译原则

```
不是禁止用"分析""生成"这类词——是把"分析什么"改成"用户能得到什么"。
"分析特征" → "看到你的线条里藏着怎样的你"
"生成报告" → "生成专属解读"
```

## 标准4: 承诺对齐（不说还没做到的事）

**"语义搜索"关键词能搜就行。"智能推荐"至少要有个排序算法。功能到了再升级命名。**

### ✅ 标杆

| 准确命名 ✅ | 过度承诺 ❌ | 为什么 |
|-----------|-----------|--------|
| "项目知识索引" | "语义知识图谱" | 当前实现=关键词grep。"语义"暗示向量检索/NLP——还没做到。 |
| "AI 趣味解读" | "AI 科学诊断" | 产品定位=趣味共鸣，不是临床心理学。 |
| "认真但不较真" | "专业人格分析" | 诚实面对产品定位——不假装自己是心理学工具。 |

---

## 标准5: 有品牌声音（不是任何产品都能用的通用语）

**品牌声音 = 拟人化(AI像一个在认真看掌纹的朋友) + 自嘲(不端着) + 不恐吓(共鸣不靠制造焦虑)。**

### ✅ 标杆

| 文案 | 品牌声音特征 |
|------|------------|
| "每条线都在认真看——第一次见面，它想认真一点" | 拟人化(AI是"它")+温和自嘲(第一次见面有点慢) |
| "你不是冷，你是慢热且挑人。" | 自嘲+洞察。不评价"好/不好"，只说"是什么"。 |
| "AI 走神了——再试一次？" | 拟人化(走神)+不甩锅(不说是用户的错) |

### ❌ 反例

| 文案 | 问题 |
|------|------|
| "请耐心等待" | 任何等待场景都能用——零品牌印记 |
| "网络异常，请检查连接后重试" | 系统报错语言——用户感受是"产品把我当IT" |

## 快速自检清单

提交任何用户可见文案前，花30秒过一遍：

```
□ 1. 有具体的锚点吗？    （去掉形容词后，还能看出说的是谁吗？）
□ 2. 温度匹配场景吗？    （首页温/报告热/错误幽默——对吗？）
□ 3. 用户能看懂吗？      （找一个非工程师读一遍——他懂了没？）
□ 4. 承诺有实现吗？      （名字说的功能，代码做到了吗？）
□ 5. 有品牌声音吗？      （换成竞品名字，这段文案还能用吗？如果能→没品牌声音）
```
