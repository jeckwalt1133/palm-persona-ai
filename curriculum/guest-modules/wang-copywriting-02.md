## 评分标准 (Grading Rubric)

本模块练习按4级评分，与毕业阶梯对齐：

| 等级 | 标准 | 判断依据 |
|------|------|---------|
| **不合格** | 无法区分L1/L2/L3，改写版本与场景错配 | 练习1诊断错2题以上；或练习2改写版本放到错误场景 |
| **L1 复现级** | 能正确诊断情感分层，能为3个场景写适配文案 | 练习1全对；练习2至少3个场景改写正确 |
| **L2 应用级** | 能调整情感温度，能判断"过度"并降温 | L1基础上+练习3至少2条打到L3+温度标注准确 |
| **L3 创作级** | 独立产出完整6场景矩阵，温度控制精准，有自评反思 | L2基础上+自评指出至少2处"还不够好"并说明理由 |

## 安全红线 (Safety Notes)

以下4条文案安全铁律来自项目合规体系（对齐 `palm-compliance` Skill），本模块所有练习必须遵守：

1. **不说命运** — 禁用"这就是你的命""你注定……""天命如此"等话术。文案解释特质，不定义命运。
2. **不说诊断** — 禁用"你有XX倾向""你可能患有XX""你的XX指数偏低"。我们做人格共鸣，不做临床判断。
3. **不说绝对** — 禁用"你永远……""你从来……""你注定……"。人的复杂性不允许绝对断言。
4. **不说恐吓** — 禁用"不改的话你会……""继续这样你会失去……"。共鸣靠理解，不靠制造焦虑。

**温度越界检查**：任何≥7°的文案，提交前必须回答三个问题：
- 如果用户不是这种感觉，这段话会让ta不舒服吗？（会→降温度）
- 去掉形容词之后，这段话还有信息量吗？（没有→补证据）
- 另一个AI实例读这段话，会觉得被理解还是被操纵？（操纵→重写）

# 模块2：情感分层与场景化文案 — 把"感动"拆成螺丝

## 学习目标

完成本模块后，你能够：
1. 把一段文案的情感拆解到具体层次，说出它触动了哪一层（不再是"感觉好"三个字）
2. 针对同一段内容，为6种不同分享场景写出差异化文案
3. 用情感温度计校准文案的力度——知道什么时候该收、什么时候该放

## 前置知识

- 完成模块01：文案共鸣基础（3层共鸣+5人格×分享动机矩阵）
- 至少写过一次分享文案并在 notebook 中有记录

## 为什么需要这个模块

模块01教你判断"有没有共鸣"。但实际写文案时，你会发现一个更棘手的问题：

```
"我知道这段文案有共鸣……但能不能再好一点？"
"这条文案朋友圈发合适，但私聊发就太尬了。"
"我写了3个版本，但不知道哪个最好。"
```

这三个问题的根因是同一个：**你对情感的操作颗粒度不够**。"感动"是一个麻袋，里面装着被理解、被看见、释然、窝心、惊喜、同理、心疼……你不能每次都从麻袋里掏"感动"出来。这个模块教你把它们拆开用。

## 第一部分：情感分层模型

### 核心框架：四级情感深度

```
Level 1 — 认知层（说得对）
  ↓
Level 2 — 情感层（它懂我）
  ↓
Level 3 — 价值层（原来这不是我的错）
  ↓
Level 4 — 行动层（我要发给XX看）
```

**不是每段文案都需要打到第4层。但每一层失守，后面的全部作废。**

**术语映射**：本模块的L1-L4与Task Prompt的三层术语一一对应：
- **表层情绪** = L1+L2（准确描述+说出感受）
- **中层共鸣** = L3（给解释+释然感）
- **深层价值观** = L4（帮用户表达+行动驱动）

### L1 认知层：准确描述，让用户点头

**触发机制**：用户看到文案 → 与自己已知的自我认知对照 → 确认"确实如此"

**判断标准**：
- 用户读完会不会说"嗯，没错"？
- 换一个人的报告，同样文案还能用吗？（如果能，说明太泛）
- 有没有引用具体的行为、偏好或选择？

**好的L1 vs 坏的L1**：

| 坏的L1（泛到能套任何人） | 好的L1（具体到让用户对号入座） |
|---|---|
| 你是一个认真的人 | 你把手机相册按年份分文件夹，截图单独一个 |
| 你有时会想太多 | 别人一句'你还好吗'，你已经在脑子里写了500字回复的草稿 |
| 你重视朋友关系 | 你记得上次聚会谁说了什么，三个月后还会拿出来引用 |

**L1 不是低层级，是所有共鸣的基础。L1 松了，后面的情感全是空中楼阁。**

### L2 情感层：说出感受，让用户卸防

**触发机制**：用户看到文案 → 发现自己没说出口的感受被精准表达 → 产生"被理解"的情感反应

**判断标准**：
- 文案是在描述"用户的处境"，还是在描述"用户处境引发的情绪"？
- 用户读完会不会沉默几秒？
- 是否有温和的"刺痛感"？（不是攻击性刺痛，是"你怎么知道"的那种）

**L1 vs L2 的区分练习**：

```
L1: "你习惯在开口前把要说的话完整过一遍"           ← 描述行为，没错
L2: "你不是不爱说话。你是太在乎说出来的是不是对的。"  ← 说出感受，扎心

L1: "你对细节有很高的要求"                           ← 陈述特征
L2: "不是挑剔。是那些粗糙的东西会让你不安——你不说，但你在忍。"  ← 说出情绪
```

**L2最容易犯的错误：跳过L1直接打情绪。**

```
❌ 跳级L2（没有事实锚点）: "你比任何人都懂什么是孤独。"
   → 用户反应："呃……是吗？"（没有具体行为锚定，情绪落空）

✅ 有L1锚点的L2: "你聚会时突然安静的那几秒，别人以为你走神了——
   其实你刚观察到一个细节，在犹豫要不要说出来。因为说出来可能冷场。"
   → L1(行为观察)→L2(被理解感)，用户点头后自然卸防
```

### L3 价值层：给解释，让用户释然

**触发机制**：用户看到文案 → 发现自己长期困惑/自我批评的事有了合理的解释 → 产生"原来不是我不好"的释然感

**判断标准**：
- 文案有没有把"症状"和"原因"连起来？
- 用户读完会不会感到释然或解脱？
- 有没有提供一个新的框架来解释用户的旧经验？

**这是产品最深的护城河。L1+L2 用户觉得准，L3 用户觉得值。**

```
L2（说出感受）: "你常常在争论中退让，不是因为没观点。"
L3（给解释+释然）: "你常常在争论中退让，不是因为没观点。
    你比别人更快算出'这场争论赢了对谁有什么好处'——
    然后发现不值得。这不是软弱，是你在乎的事情不在输赢上。"
    ↑ 把"退让=软弱"重新定义为"退让=算得通透"
```

**L3 的五种深度解释模式**：

| 模式 | 原理 | 示例触发词 |
|------|------|-----------|
| **重新定义** | 把负面特征用正面框架理解 | "这不是X，这是Y" |
| **溯源归因** | 把当下的感受连到过去的经历 | "因为你经历过……所以你现在……" |
| **普遍化** | 让用户知道不是自己一个人这样 | "很多人都有这种感觉，只是不说" |
| **功能化** | 解释这个特质曾经/正在起什么作用 | "你的敏感曾经保护过你" |
| **预见化** | 基于特质预测用户可能面临的矛盾 | "这让你在某些场合……而在另一些场合……" |

**L3 的红线（绝对不能碰）**：
1. **不说命运** — "这就是你的命"是辱骂，不是解释
2. **不说诊断** — "你有轻度抑郁倾向"不是你能给的结论
3. **不说绝对** — "你永远/从来/注定"是语言暴力
4. **不说恐吓** — "不改的话你会孤独终老"是邪教，不是产品

### L4 行动层：帮用户表达，让用户分享

**触发机制**：用户已经经历L1(点头)→L2(卸防)→L3(释然) → 产生"想让某个人也看到这段话"的冲动

**判断标准**：
- 用户读完会不会打开微信？
- 分享后，接收方看到卡片会不会也想测？
- 分享出去的内容，能不能帮用户省掉1000字？

**L4 不是加一个"分享给朋友"按钮。L4 是文案本身具备"帮用户对别人说了他想说但不会说的话"的属性。**

```
坏L4: "分享到朋友圈看看别人怎么评价你吧！"     ← 索要分享，不提供价值
好L4: "你的报告说'你更擅长在事后理解情绪'——     ← 帮用户表达了自我
      把这句话截图发给上次被你惹恼的朋友。
      配字：'报告说我的反射弧有点长……但我在意。'"
```

**L4 的三种分享驱动力**：

| 驱动力 | 用户心理 | 文案策略 |
|--------|---------|---------|
| **自我表达** | "这就是我，发给别人省得我解释" | 文案本身是精准的自我介绍 |
| **关系修复** | "我想说但不知道怎么开口" | 文案充当中间人/缓冲带 |
| **社交货币** | "这条挺有意思，发出来显得我有品" | 文案有信息增量或独特视角 |

## 第二部分：场景化文案

### 核心认知：文案不存在"最好"的版本，只存在"最适配当前场景"的版本

同一句人格洞察，在朋友圈发、私聊发、群聊发的文案完全不同。不是因为洞察变了，是场景的**信任半径、情感承载量、注意力时长**不同。

### 六大场景矩阵

| # | 场景 | 信任半径 | 注意力 | 情感承载 | 文案策略 |
|---|------|---------|--------|---------|---------|
| 1 | **朋友圈晒图** | 中（好友可见，但不全是密友） | 2-3秒 | 中低 | 一句话钩子+好奇心缺口 |
| 2 | **私聊转发** | 高（1v1，已知关系） | 10-30秒 | 高 | 情感打点+关系锚定 |
| 3 | **群聊讨论** | 低（多人，半熟人） | 3-5秒 | 低 | 话题钩子+低门槛参与 |
| 4 | **收藏自省** | 无（只给自己看） | 不限 | 极高 | 深度共鸣+反思空间 |
| 5 | **截屏传播** | 低（可被转发到任何地方） | 1-2秒 | 低 | 强力钩子+品牌水印 |
| 6 | **二次传播** | 极低（三手传播） | 0.5秒 | 极低 | 极度简练+视觉锚点 |

### 场景改写示例：同一洞察 × 6种场景

以下示例使用同一条核心洞察："你更擅长在事后理解情绪，而非在当下表达。"

**原始洞察（L2级）**：
> 你总是离开之后才想清楚当时该说什么。

| # | 场景 | 改写后文案 | 为什么这样改 |
|---|------|-----------|-------------|
| 1 | **朋友圈** | "有些人的嘴是实时系统，有些人的嘴要跑一个批处理——你是后者。" | 轻量自嘲+技术比喻有趣，好友看了会心一笑而不觉得你在暴露弱点。2秒可读。 |
| 2 | **私聊** | "报告说我的情感CPU是离线渲染——当时没反应，但每一帧都是认真算的。上次跟你说'没事'，其实是还没算完。这就是我，你多担待。" | 1v1场景可以暴露脆弱，用具体回忆做锚点。语气亲密自然，不端着。结尾"你多担待"给接收方一个明确的情绪回应点。 |
| 3 | **群聊** | "测了一个报告，说我'吵架水平严重受制于内存延迟'——意思是吵完3小时才想起最佳反驳。有人同款吗？" | 群聊需要话题钩子+低门槛参与。"有人同款吗"降低回复成本，谁都能接一句"哈哈我也是"。 |
| 4 | **收藏** | "你不是反应慢。你的大脑在说出口之前，已经把这句话在5个人的视角里跑了一遍——确认不会伤到任何人之后，时机已经过了。这份审慎不是弱点，是你比大多数人都先学会了'言语的重量'。" | 无字数限制，可以做完整L1→L2→L3→L4链路。语气可以更沉、更书面，因为读者是在安静独处时读的。 |
| 5 | **截屏** | "我的大脑：\n▮▮▮▮ 实时反应\n▮▮▮▮ 事后反思\n▮▮▮▮ 凌晨三点突然想到最佳回复\n—— 掌心人格局" | 视觉锚点强（进度条+反差），截屏传播自带品牌水印。不需要解释，画面本身即内容。 |
| 6 | **二次传播** | "吵完架的第二天凌晨3点才想到'其实我该这么说'——如果你的大脑也这样，this is for you。" | 极简触动点。三手传播的人不一定会去测，但会截图发给另一个"同款"朋友。 |

**关键教训**：
- 朋友圈发私聊版的深度文案 → 太沉重，点赞数低。不是文案不好，是场景错配。
- 私聊发截屏版 → 对方觉得你在敷衍。不是文案不好，是场景错配。
- **文案的质量 = 内容共鸣 × 场景适配度。一个乘数为零，结果为零。**

### 场景选择决策树

```
你要为一段洞察写分享文案。先问三个问题：
  
  Q1: 用户最可能在什么场景下分享它？
      ├─ 一个人安静看报告 → 场景4（收藏自省）为主，场景1（朋友圈）为辅
      ├─ 两个人正在聊天 → 场景2（私聊）为主
      └─ 一群人热闹讨论 → 场景3（群聊）为主
  
  Q2: 接收方的信任关系是什么？
      ├─ 亲密关系（伴侣/挚友） → 情感承载可以高，允许展示脆弱
      ├─ 一般好友 → 轻量自嘲/趣味解读，不要过度暴露
      └─ 半熟人/陌生人 → 只展示社交货币面，不暴露情感面
  
  Q3: 用户想借这条文案达到什么？
      ├─ 表达自我（"这就是我"） → 走L2深度路线
      ├─ 修复关系（"上次是我不好"） → 走L3释然路线
      ├─ 展示品味（"这个挺有意思"） → 走轻量趣味路线
      └─ 发起对话（"你也测一个？"） → 走互动邀请路线
```

## 第三部分：情感温度计

### 工具定义

同一个洞察，情感温度可以调。知道调到什么温度，是高级能力。

```
冷  0-2  客观陈述       "分析显示你的情绪表达倾向于延迟处理"
温  3-5  温和共鸣       "你是个思考比表达快半拍的人"
热  6-8  强烈共情       "每一次沉默背后，都有一句反复排练却没有说出口的话"
烫  9-10 过度煽情/越界  "你是被世界忽略的灵魂，沉默是你最后的尊严"
                          ↑ 这是PUA，不是共鸣。永远不要到这一档。
```

### 温度校准示例

同一洞察（"倾向于延迟情绪表达"），不同温度：

| 温度 | 文案 | 适用场景 | 风险 |
|------|------|---------|------|
| **冷 2°** | "情绪表达模式：延迟处理型。平均反应间隔高于常模1.8个标准差。" | 学术报告、数据摘要 | 准确但没人想分享 |
| **温 3°** | "你是个想清楚了才说的人——有时候想得太清楚，就错过了说的时机。" | 朋友圈、报告主体 | 安全区，不容易出错 |
| **温 5°** | "你不是没感觉。你是每次有感觉时，大脑自动弹出一个'确认发送？'的窗口——然后你点了取消。" | 私聊、收藏自省 | 开始有共鸣，需要准确的L1锚点 |
| **热 7°** | "你一生中被误解最深的时刻，不是因为说错了，是因为什么都没说。而那些没说出口的话，每一条你都记得。" | 深度报告卡片、"被误解最深点"模块 | 共鸣上限，再往上就是煽情 |
| ~~烫 10°~~ | ~~"你是沉默的囚徒，被自己的谨慎困在原地。"~~ | 永不使用 | 攻击用户，一票否决 |

**温度校准铁律**：
1. **默认用温（3-5°）**。温不会出错，冷没价值，热需要精准的L1锚点支撑。
2. **每升1°需要2倍的L1证据支撑**。3°只需要"你倾向于延迟表达"，7°需要具体到某个行为、某个时刻、某个细节。
3. **6°以上必须经第二人审查**。你自己写的热文案，你自己很难判断是不是过度了。
4. **禁止写烫（9-10°）文案**。烫不是共鸣，是操纵。判断标准：如果用户不是这种感觉，文案会不会让ta不舒服？会→烫。

### 降温技巧（当你发现文案太热时）

```
热： "每一次沉默背后，都有一句反复排练却没有说出口的话"
       ↓ 加自嘲
温： "吵完架3小时后想到最佳反驳——恭喜你，你的大脑是离线渲染引擎"
       ↓ 加数据视角
冷： "情绪表达延迟型：82%的人事后能清晰回忆当时该说的话，但只有23%的人当下说出口"
```

### 升温技巧（当你需要更打动人时）

```
冷： "你倾向于在事后处理情绪信息"
       ↓ 加具体场景
温： "你总是在离开饭局后，才想清楚刚才那句话的真正意思"
       ↓ 加情绪描述
热： "你不是听不懂弦外之音。恰恰相反——你听懂了，只是没办法在那个瞬间回应。
      你需要独处的时间来'翻译'那些细微的信号。而这让你错过了很多本该接住的瞬间。"
```

## 常见问题

**Q: L3 和 L2 的边界有时候很模糊，怎么判断？**
L2 是"我的感受被说出了"。L3 是"我的感受被解释了"。区别在于是否提供了一个新的因果框架。L2 说"你很累"，L3 说"你很累是因为……"——那个"因为"就是L3。

**Q: 所有文案都必须打到 L3 吗？**
不。场景1（朋友圈）和场景3（群聊）通常打到 L2 就够了，打 L3 反而太重。场景2（私聊）和场景4（收藏自省）才需要 L3。记住：**选对层 = 选对力道。过分比不够更糟。**

**Q: 怎么判断自己写的文案是不是"烫"了？**
两个测试：① 读给另一个AI实例听，问它"这段文案有没有操纵感？"② 换位思考——如果有人对你说这句话，你是觉得被理解了，还是觉得被冒犯了？如果是后者，它就是烫的。

**Q: 6个场景我记不住怎么办？**
不需要全记。开发时先写场景4（收藏自省版），这是最完整、最不受限制的版本。然后从它出发做减法——朋友圈版砍到1句钩子，私聊版加上关系锚点，截屏版转成视觉语言。**从最长的开始写，往短了砍。**
