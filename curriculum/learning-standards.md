# "学会"三级标准

## 为什么需要这个

PROTOCOL.md定义了学生"学习研究、主力开发"——但从未定义"学会了"意味着什么。

V4-V6期间学生名义上"完成了7个产出等级"，但实际上：
- 部分产出是老师代劳的
- 没有人问过"学生能不能独立再做一次"
- "完成"的判断标准是"文件存在"，不是"能力具备"

这个文档定义三级标准，适用于**每一个知识单元**（一门课、一个任务、一个技术领域）。

---

## 三级标准

### L1 复现级 — 能跑通

**定义**：在有人提供明确步骤的情况下，能独立执行并得到预期结果。

**判断标准**：
- [ ] 能在干净环境中独立搭建项目（npm install + 启动）
- [ ] 能跑通已有代码并得到预期输出
- [ ] 能解释每步在做什么（不是死记命令）
- [ ] 遇到常见错误能自己解决（不需要问老师）

**证明方式**：
- 从零开始，独立操作
- 记录操作步骤和遇到的坑（笔记）
- 老师抽查："把这个环境删了，重新搭建"

**代表能力**：follow — 跟着能做

**对应毕业等级**：1.1 入门生

---

### L2 改进级 — 能优化

**定义**：在理解现有实现的基础上，能独立做可度量的改进。

**判断标准**：
- [ ] 能指出当前实现的问题（不只是"觉得不好"，能说"哪里不好、为什么不好"）
- [ ] 能提出改进方案并自己实现
- [ ] 改进效果可度量（跑得更快/准确率更高/代码更简洁/测试覆盖更多）
- [ ] 修改后不引入回归（测试通过）

**证明方式**：
- 改进前后的对比数据（基准线→优化后）
- 改进方案的设计文档（为什么选这个方案）
- 代码审查通过（无重大设计问题）

**代表能力**：improve — 能独立改进

**对应毕业等级**：1.2 修学者 → 2.1 研读者

---

### L3 教学级 — 能带人

**定义**：能把自己的理解结构化地传授给另一个人（或AI实例），让ta达到L1级。

**判断标准**：
- [ ] 能写出清晰的教学材料（不是笔记，是教材——目标/前置/步骤/验收）
- [ ] 能预判学习者可能遇到的坑并提前说明
- [ ] 学习者独立操作成功（不需要教学者代劳）
- [ ] 能回答学习者的"为什么"（不仅知道How，知道Why）

**证明方式**：
- 带一个学徒完成同一知识单元的L1级任务
- 学徒的独立操作笔记（证明没有代劳）
- 学徒的反馈："哪里讲得清楚，哪里没听懂"

**代表能力**：teach — 能教会别人

**对应毕业等级**：2.3 发表者 → L0.2 导师

## 使用规则

### 规则1：每个任务标注目标级别

在Task Prompt中明确：这个任务是期望学生达到L1/L2/L3？

```
例：
- "看懂现有Mock管线" → L1（能跑通+能解释）
- "设计Sharp替代方案" → L2（能改进+有对比数据）
- "带学徒搭建开发环境" → L3（能教别人）
```

### 规则2：不跳级

同一知识单元必须先L1再L2再L3。不允许：
- 没跑通过就提改进方案（L1未到就做L2）
- 没做过改进就带学徒（L2未到就做L3）

但不同知识单元可以并行。例如：在AI Provider领域达到L2的同时，在合规领域达到L1。

### 规则3：复检机制

L1/L2/L3的判定不是永久有效的。
- L1判定后30天无相关实践 → 自动退回未验证
- L2判定后60天无相关实践 → 自动退回L1
- L3判定后90天无相关实践 → 自动退回L2

这叫"学会的保质期"——不用就会退化。

### 规则4：辩论权

学生对老师的判定有异议时 → 触发三角批判（老师+学生+1位客座讲师）
- 学生举证：我能独立完成X
- 老师质证：上次你需要帮助才能完成
- 客座裁决：现场出题验证

## 与毕业矩阵的映射

| 毕业等级 | 对应的能力级别 | 说明 |
|---------|-------------|------|
| 1.1 入门生 | 5个核心知识单元达到L1 | 能跑通基础 |
| 1.2 修学者 | 10个知识单元达到L1 + 3个达到L2 | 有改进能力 |
| 1.3 毕业生 | 1个完整项目达到L2（从设计到交付） | 独立交付 |
| 2.1 研读者 | 1个研究领域达到L2（能批判分析） | 独立研究 |
| 2.2 实验者 | 1个实验达到L2（能设计+执行+分析） | 独立实验 |
| 2.3 发表者 | 1个实验达到L3（能写出可复现文档） | 可被引用 |
| 3.1 交付者 | 1个功能模块达到L2 | 独立交付 |
| L0.1 助教 | 1个知识单元达到L3（带1个学徒到L1） | 教学验证 |
