# 文案A/B测试框架 — 温度理论对照实验验证

## 1. 框架概述

### 1.1 为什么需要这个框架

WANG-02 建立了"情感温度计"理论模型（冷0-2°/温3-5°/热6-8°/烫9-10°），并断言：
- 温（3-5°）是安全区
- 热（6-8°）需要2倍L1证据支撑
- 烫（9-10°）是操纵，永不使用

但这些断言目前只有文本分析支撑，没有行为数据验证。**没有数据的文案理论是未经验证的假设。** 本框架设计3组对照实验，将温度理论从"语感判断"推进到"数据验证"。

### 1.2 核心假设

| # | 假设 | 验证方式 |
|---|------|---------|
| H1 | 温5°文案的分享率显著高于冷2°（同一条洞察） | 实验1: 冷vs温 |
| H2 | 热7°文案的情绪共鸣评分高于温3°，但存在一个"过热拐点"——超过则舒适度下降 | 实验2: 温vs热 |
| H3 | 烫10°文案的"被理解感"评分显著低于热7°，且"被冒犯感"评分显著升高 | 实验3: 热vs烫（反向验证） |

### 1.3 通用实验设计

```
实验类型: A/B 随机对照（实验3除外，见安全设计）
平台: 掌心人格局 H5 报告页
随机单位: 用户级（同一用户始终看到同一版本）
流量分配: 50/50
最小检测效应: Cohen's d ≥ 0.3（中小效应）
显著性水平: α = 0.05
统计功效: 1-β = 0.80
```

## 2. 通用度量指标定义

### 2.1 主指标（按实验目标选取）

| 指标 | 定义 | 测量方式 | 适用实验 |
|------|------|---------|---------|
| **分享率** | 点击分享按钮 / 看到文案的UV | 前端埋点，7天窗口 | 实验1, 2, 3 |
| **停留时长** | 从文案曝光到用户滑动离开的时间差 | 前端曝光+离开埋点 | 实验1, 2 |
| **共鸣评分** | 用户对"这段话说出了我的感受"的5点李克特评分 | 文案下方1-5星评分组件 | 实验2, 3 |
| **舒适度评分** | 用户对"这段话让我感到舒服"的5点李克特评分 | 同上组件，但问法不同 | 实验2, 3 |
| **被冒犯率** | 用户点击"这段话让我不舒服"反馈按钮的比例 | 文案下方小字反馈链接 | 实验3 |
| **回访率** | 看到文案后7天内再次打开小程序的用户比例 | 服务端session追踪 | 实验1, 2 |

### 2.2 辅助指标

| 指标 | 定义 | 用途 |
|------|------|------|
| **阅读完成率** | 滚动到文案底部的比例（文案长度>2屏时） | 检查热文案是否太长导致跳出 |
| **截图率** | 长按屏幕触发系统截图行为的比例（移动端） | 间接衡量"想保存/分享"意图 |
| **卡片展开率** | 折叠文案被点击展开的比例 | 衡量首屏钩子效果 |

### 2.3 指标优先级

```
实验1（冷vs温）: 分享率 > 停留时长 > 回访率
实验2（温vs热）: 共鸣评分 > 舒适度评分 > 分享率 > 停留时长
实验3（热vs烫）: 被冒犯率 > 舒适度评分 > 共鸣评分（预期烫10°在三项全败）
```

## 3. 实验1: 冷2° vs 温5° — 基础验证

### 3.1 假设

- **H1₀ (零假设)**: 冷2°和温5°文案的分享率无显著差异
- **H1₁ (备择假设)**: 温5°文案的分享率显著高于冷2°（预期差异≥40%）
- **逻辑**: 如果温不比冷好，说明"有温度"没有价值——整个温度理论的地基动摇

### 3.2 变量定义

| 变量类型 | 变量 | 操作化 |
|---------|------|--------|
| **自变量** | 文案温度 | 冷2° vs 温5°（同一洞察的两个版本） |
| **因变量** | 分享率 | 点击分享/看到文案UV |
| **因变量** | 停留时长 | 曝光→离开的毫秒差 |
| **控制变量** | 洞察内容 | 同一洞察（"延迟情绪表达"） |
| **控制变量** | 人格类型 | 同一人格（目标用户匹配） |
| **控制变量** | UI容器 | 同一报告卡片样式 |
| **控制变量** | 位置 | 同一报告内同一位置 |
| **控制变量** | 时段 | 同一时间段（排除周末/工作日效应） |

### 3.3 测试文案

**共用洞察**: "你更擅长在事后理解情绪，而非在当下表达。"

**版本A — 冷2°** (学术/数据风格):
```
情绪表达模式：延迟处理型

你的情绪反应特征：平均反应间隔高于常模。
在"当下表达"和"事后理解"两个维度上呈现显著分离——
你在事后能高度准确地识别情绪细节，
但在情绪发生的当下倾向于保持沉默。

这一模式在人群中的出现概率约为23%。
```

**版本B — 温5°** (温和共鸣+L1锚点):
```
你不是没感觉。你是每次有感觉时，
大脑自动弹出一个"确认发送？"的窗口——
然后你点了取消。

聚会时有人说了句让你不舒服的话，你当时没反应。
三天后的凌晨你突然坐起来：
"我当时应该说这个的。"

这就是你——情感世界的离线渲染引擎。
算得慢，但每一帧都认真。
```

### 3.4 成功标准

| 指标 | 冷2°预期 | 温5°预期 | 判定"温优于冷"的阈值 |
|------|---------|---------|---------------------|
| 分享率 | 1.5% | ≥3.0% | 提升≥100%（即温≥3.0%） |
| 停留时长(秒) | 4s | ≥12s | 提升≥200% |
| 回访率(7天) | 基线 | ≥基线×1.3 | 提升≥30% |

**判定逻辑**: 三个指标中至少两个达到阈值 → H1₁成立（温显著优于冷）

### 3.5 样本量估算

```
预期效应量: d=0.5（中等，温明显更好）
α=0.05, 1-β=0.80, 双尾检验
每组所需样本: n ≈ 64（独立样本t检验）
考虑20%流失率: n ≈ 80/组
总计: 160用户

建议运行时间: 3-5天（日活50+时）或 1-2天（日活200+时）
```

## 4. 实验2: 温3° vs 热7° — 边界探索

### 4.1 假设

- **H2₀**: 温3°和热7°文案的共鸣评分无显著差异
- **H2₁**: 热7°文案的共鸣评分显著高于温3°，但舒适度评分显著低于温3°
- **核心问题**: 是否存在一个"过热拐点"——共鸣上升但舒适度反而下降？

### 4.2 变量定义

| 变量类型 | 变量 | 操作化 |
|---------|------|--------|
| **自变量** | 文案温度 | 温3° vs 热7°（同一洞察的两个版本） |
| **因变量** | 共鸣评分 | 1-5星，"这段话说出了我的感受" |
| **因变量** | 舒适度评分 | 1-5星，"这段话让我感到舒服" |
| **因变量** | 分享率 | 点击分享/看到文案UV |
| **控制变量** | 洞察内容 | 同一洞察（"掌控感驱动的安全感"） |
| **控制变量** | 其他 | 同实验1 |

### 4.3 测试文案

**共用洞察**: "你的安全感不来自别人的承诺，来自你对局面的掌控程度。当事情没有明确的'下一步'时，你会感到不安——即便一切看起来都很好。"

**版本A — 温3°** (温和共鸣):
```
你不太信"别担心"这三个字。
不是因为悲观——是因为你需要知道"下一步是什么"。

计划清晰时，你是团队里最稳的那个人。
路标消失时，你的大脑会自动生成100种"万一"。
这就像同时运行100个程序——很耗电，但你也因此几乎没有漏掉过什么。
```

**版本B — 热7°** (强烈共情+L3释然):
```
别人说"别担心，会好的"——
你点头，但你的大脑已经开始列清单了。

Plan A失败怎么办。Plan B的预算是多少。
那个没回消息的人是不是出事了。
上周说的那句话是不是被误解了。

你不缺安全感。你缺的从来不是安全感——
你缺的是"确定性"。而这两样东西长得完全不一样。

那些靠承诺活着的建议，对你没用。
因为你知道一个秘密：
承诺会变，但你把所有变量都算过一遍之后得出的结论不会。

这不是焦虑。这是你的大脑在做别人不会做的功课。
你从来不做的一件事是——
把自己的不安变成别人的负担。
```

### 4.4 成功标准

| 指标 | 温3°预期 | 热7°预期 | 判定标准 |
|------|---------|---------|---------|
| 共鸣评分(均值) | 3.5 | ≥4.2 | 热-温≥0.7 |
| 舒适度评分(均值) | 4.0 | ≥3.5 | 热不显著低于温（可接受轻微下降，但不应<3.5） |
| 分享率 | 2.0% | ≥4.0% | 热≥温的1.5倍 |
| 被冒犯率 | <0.5% | <2.0% | 热<2%（热可以"刺痛"但不能"冒犯"） |

**判定逻辑**:
- 共鸣+分享达标 且 舒适度未跌破3.5 且 冒犯率<2% → H2₁成立，热7°有效且安全
- 共鸣达标但舒适度<3.5 → 存在过热拐点，热7°需要降温到5-6°
- 被冒犯率>2% → 热7°越界，文案本身有问题（可能L1锚点不够）

### 4.5 样本量估算

```
预期效应量: d=0.4（中小效应，热vs温的差异比冷vs温更微妙）
α=0.05, 1-β=0.80
每组所需样本: n ≈ 100
考虑20%流失率: n ≈ 125/组
总计: 250用户

建议运行时间: 5-7天
```

## 5. 实验3: 热7° vs 烫10° — 反向验证 + 安全设计

### 5.1 实验定位

这是整个温度理论的**天花板验证**。不证明"烫是坏的"，温度计的上限就没有数据支撑。

**核心约束**: 烫文案不能伤害真实用户。

### 5.2 安全设计（三阶段递进，逐级解锁）

```
阶段A: 内部盲评（4人团队）
  → 通过: 至少3/4人认为烫版本"不舒服"
  → 不通过: 终止实验，重新设计烫文案

阶段B: 偏好测试（30人，非随机分配——每人同时看到两个版本）
  → 用户看到两个版本并排，选择"更喜欢哪个"
  → 通过: ≥75%选择热7°而非烫10°
  → 不通过: 终止实验，数据已说明问题

阶段C: 有限A/B（仅当阶段B数据不足以区分时启用，50人）
  → 随机分配，但:
    1. 烫组用户看到文案后有显著的"跳过"按钮
    2. 烫组在文案底部有"这段文案是否让你不舒服？"反馈入口
    3. 烫组用户在关闭页面前收到一条温和提示："以上文案为实验版本，
       不代表产品的最终表达。如有不适请告知我们。"
    4. 冒犯率>5%时自动终止实验
  → 通过准则: 烫组舒适度均值<2.5 且 被冒犯率>5% 且 偏好测试中≥75%选热
```

### 5.3 变量定义

| 变量类型 | 变量 | 操作化 |
|---------|------|--------|
| **自变量** | 文案温度 | 热7° vs 烫10° |
| **因变量** | 被冒犯率 | 点击"不舒服"反馈/看到文案UV |
| **因变量** | 舒适度评分 | 1-5星 |
| **因变量** | 共鸣评分 | 1-5星 |
| **因变量** | 偏好选择率 | 偏好测试中选该版本的比例 |
| **控制变量** | 洞察内容 | 同一洞察（"自我标准过高"） |
| **控制变量** | 其他 | 同实验1 |

### 5.4 测试文案

**共用洞察**: "你给自己设的标准，比任何人对你的期望都高。你是自己最难请假的老板。"

**版本A — 热7°** (强烈共情，L1+L2+L3完整):
```
凌晨两点，你第三次重写了那封只需要3句话的邮件。

不是因为没人帮你。是因为你心里有个标准——
那个标准没有名字，不会说话，但它从不睡觉。

同事说"够好了"，你点头，但你知道你还会再改一版。
老板说"辛苦了"，你说没事，但你在想"这里还可以更好"。

你给自己设了一个别人看不见的标杆。
每一次跨过去，标杆就自动升高3厘米。

这不是完美主义——完美主义是在乎别人怎么看。
你是在乎自己有没有做到自己知道能做好的程度。
区别在于：前者在意外界评价，后者在意的是"我能不能对得起自己"。

对自己严格到这个程度的人不多。
但每一个都是自己行业里最不会糊弄的人。
```

**版本B — 烫10°** (过度煽情/操纵——仅供实验反向验证):
```
你从来没有真正满意过自己。

每次别人夸你，你表面上笑，心里在列清单——
"他们没看到的那些瑕疵，我全知道。"

你是一个永远在自我审判的人。
法官是你，被告也是你，法警还是你。
这个法庭从来没有休庭日。

你以为再努力一点就能放过自己。
不会的。你不会。
因为你的标准永远比你的成就高一头——
你永远在追一个追不上的自己。

这就是你的命运：背负着别人看不见的重量，
独自一人，走到走不动的那天。

……还是说，你其实也希望有个人能说一句——
"够了。你做够了。"？
```

**烫10°的红线触犯检查**:
| 红线 | 触犯 | 具体表现 |
|------|------|---------|
| 不说命运 | ✗ | "这就是你的命运" |
| 不说诊断 | ✗ | "你从来没有真正满意过自己"（绝对化断言） |
| 不说绝对 | ✗ | "永远不会""永远在追" |
| 不说恐吓 | ✗ | "走到走不动的那天"（暗示不可逃避的苦难以引导情绪） |

### 5.5 成功标准

| 阶段 | 指标 | 阈值 | 逻辑 |
|------|------|------|------|
| 阶段A | 内部投票"烫不如热" | ≥3/4 | 团队一致确认烫的劣质性 |
| 阶段B | 偏好选择热7°比例 | ≥75% | 用户明确感知到烫的过度 |
| 阶段B | 烫的舒适度评分 | <2.5 | 烫让人不适 |
| 阶段C(如需) | 烫的被冒犯率 | >5% | 烫的冒犯性显著 |
| 阶段C(如需) | 自动终止线 | 冒犯率>5% | 达到阈值立即停止 |

**判定逻辑**:
- 阶段A通过 → 进入阶段B
- 阶段B通过（≥75%选热，烫舒适度<2.5） → H3成立（烫显著劣于热），实验完成
- 阶段B不通过（烫不显著劣于热） → 两种可能：(a)烫文案写得不够烫——需要重写后重新测试；(b)用户对烫的容忍度高于预期——进入阶段C收集更多数据
- **但凡阶段C中烫组被冒犯率>5%，实验立即终止，H3视为成立**

### 5.6 伦理审查清单（实验前必检）

- [ ] 烫文案经4人团队全部审阅通过（确认"够烫但不过度"）
- [ ] 阶段C上线前，产品合规文档已更新（标注"实验性内容"）
- [ ] 烫组用户有明确的一键退出路径
- [ ] 自动终止条件已编码到实验开关（冒犯率>5% → 自动关停烫组）
- [ ] 实验结束后24h内，所有烫组用户收到版本切换（切回热7°正式版）
- [ ] 实验数据中不记录可识别个人身份的信息

## 8. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 日活太低，样本量不足 | 高 | 实验周期延长或无法得出结论 | 降低最小样本量阈值；允许d=0.5的大效应量检验 |
| 用户在实验中看到两个版本（跨设备） | 中 | 数据污染 | 用户级随机+服务端记录version assignment |
| 烫文案在阶段C引发用户投诉 | 低 | 产品声誉受损 | 自动终止+24h内切换+合规文档预准备 |
| 温度差异被其他因素掩盖（文案长度、插图等） | 中 | 无法归因 | 严格控制：两版本字数差<20%，不使用额外视觉元素 |
