# 第8课：多模态 Agent

## 多模态 Agent 架构

```
用户输入（文本/图片/语音）
        ↓
  ┌─────────────────┐
  │   Router/Classifier │  判断输入类型
  └─────────────────┘
        ↓
  ┌──────────┬──────────┬──────────┐
  │ LLM      │ Vision   │ Audio    │
  │ (文本)    │ (图片)    │ (语音)    │
  └──────────┴──────────┴──────────┘
        ↓
  ┌─────────────────┐
  │   Fusion Layer  │  多模态融合
  └─────────────────┘
        ↓
  工具调用 / 响应生成
```

## 掌心人格局的视觉链路

```
用户上传手掌照片
  → imageCompress (前端压缩)
  → PalmFeatureMarker (特征标注)
  → 特征提取 (15+ 维度)
  → 分析引擎 → 报告生成
```

视觉在这里是**特征提取**（非 LLM 视觉），作为分析引擎的输入，不是作为 AI Vision 调用。

## 多模态融合模式

1. **Early Fusion**：在输入端合并多模态数据 → 一次性处理
2. **Late Fusion**：各模态单独处理 → 在决策层合并
3. **Hybrid**：部分早期融合 + 部分后期融合

## 工具调用（Tool-use）

```
Agent 思考 → 决定调用工具 → 执行工具 → 工具结果回传 → 继续思考
```

MCP Tools 是多模态 Agent 的"行动层"——补足 LLM 无法做的事：
- 调用外部 API
- 读取数据库
- 执行命令
- 操作文件

## 考点
- 三种多模态融合模式
- Vision / Audio 在 Agent 中的角色
- Tool-use 生命周期
- MCP Tools 作为行动层的定位
