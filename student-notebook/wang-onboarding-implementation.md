## 实现方案

### 1. 首次用户检测

```
检测逻辑: localStorage.getItem('palm_onboarding_shown')
  → null → 首次用户 → 展示弹窗，从Step 1开始
  → '1'  → 回访用户 → 跳过，直接看到首页
关闭时: localStorage.setItem('palm_onboarding_shown', '1')
```

### 2. 3步弹窗状态机

```
Step 1 (截图引导) → 点击"知道了，下一步" → Step 2
Step 2 (等待预览) → 点击"然后呢？"       → Step 3
Step 3 (啊哈预告) → 点击"开始拍手掌"      → 关闭弹窗 + 记录已展示
任意步骤         → 点击"跳过引导"          → 关闭弹窗 + 记录已展示
```

### 4. UI组件结构

```
onboarding-overlay (fixed全屏 92%暗色叠加)
└── onboarding-modal (居中卡片 max-w 560rpx)
    ├── onboarding-steps (3个步骤圆点 + 连接线)
    │   ├── step-dot 1: step-active/step-done
    │   ├── step-dot 2: step-active/step-done
    │   └── step-dot 3: step-active/step-done
    ├── onboarding-body
    │   ├── step-label ("第N步")
    │   ├── title (金色大字)
    │   └── line × 3 (白色半透明)
    └── onboarding-actions
        ├── btn-primary (金色渐变按钮)
        └── btn-skip (灰色文字链接)
```

### 5. 样式设计

- 叠加层：`rgba(12, 8, 28, 0.92)` 深紫黑92%不透明，与品牌暗色主题一致
- 活跃步骤点：金色填充 + 金色边框 + 发光阴影 `box-shadow: 0 0 24rpx rgba(255,209,102,0.15)`
- 已完成步骤点：绿色 `#06D6A0` 填充 + ✓ 图标
- 标题：36rpx 金色加粗
- 正文：26rpx 白色60%透明度，行高1.7
- 主按钮：金色渐变 `#FFD166 → #F5A623`，暗色文字
- 跳过按钮：暗灰色，hover变亮

### 6. "跳过引导"的用户体验

- 任何步骤都可以点"跳过引导，直接开始"——不给用户制造被困住的感觉
- 跳过也会记录 `palm_onboarding_shown = '1'`，下次不再弹
- 不强制看完——强制引导是反UX的。只有用户自己选择看下去的引导才有价值
