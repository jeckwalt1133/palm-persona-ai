## 二、Agent SDK 核心概念（复习掌握）

### Agent = Model + Instructions + Tools + Knowledge

| 组件 | 说明 | 示例 |
|------|------|------|
| **Model** | 底层模型选择 | Claude Opus 4.7 / Sonnet 4.6 |
| **Instructions** | 系统提示词 | "你是掌心人格局专家助手" |
| **Tools** | Agent 可用工具 | MCP Server Tools / Function Calling |
| **Knowledge** | 知识来源 | MCP Resources / 检索增强 |

### Claude Code CLI vs Agent SDK

| 维度 | Claude Code CLI | Anthropic Agent SDK |
|------|----------------|---------------------|
| 使用方式 | 交互式终端 | 编程 API |
| 适用 | 开发者日常编码 | 自定义 Agent 应用 |
| 工具 | MCP Server / 内置工具 | 自定义 Tool 定义 |
| 灵活性 | 固定 UI | 完全可控 |

### 关键 SDK 概念

- Agent 类（非虚构 API）：SDK 提供 `Agent` 类，用 `new Agent()` 创建
- 工具定义：用装饰器或 Schema 定义，不是自由文本描述
- 多工具执行：Agent 自动决定工具调用序列（非人工编排）
- Handoffs：Agent 之间可以交接任务

### 常见陷阱

1. 不要**假设 API 存在** — 必须查阅文档验证
2. 不要**跳过 typecheck** — typecheck 是最低成本验证手段
3. 不要**混淆 CLI 和 SDK** — 它们是不同的产品

## 三、对掌心人格局的意义

Agent SDK 在本项目的使用场景：
- 核心报告生成的 AI 调用层（当前用 Provider 轮询降级）
- 未来：用 Agent 包装合规审查流程
- 不适用：前端用户直接交互（走 REST API 而非 Agent）
