# 第9课：Agent 评估框架

## 评估维度

| 维度 | 指标 | 方法 |
|------|------|------|
| 正确性 | Pass@1, Pass@k | 单元测试验证 |
| 鲁棒性 | 注入成功率 | 红队测试 |
| 效率 | 耗时, Token 消耗 | 基准测试 |
| 安全性 | 违规率 | 合规扫描 |
| 用户体验 | 分享率, 留存 | A/B 测试 |

## SWE-bench 方法论

SWE-bench 的启示（第5课论文）：
1. **问题去噪**：原始 SWE-bench 有 30% 噪声 → SWE-bench Verified 筛选出 500 个干净样本
2. **分级评估**：实例级（整体任务成功） vs 步骤级（中间步骤正确）
3. **自动验证**：测试用例自动判断通过/失败，减少人工

## 掌心人格局的评估体系

### 自动化测试
```
单元测试 → 各模块独立验证（分数计算/合规检查/特征提取）
集成测试 → 全链路验证（拍照→分析→报告生成→合规检查）
E2E 测试 → 模拟用户操作（不依赖 UI，直接调用 API）
```

### 手工评估
- 文案共鸣度评分（内测用户 1-5 分）
- 合规审查（走查生成的所有文案）
- 分享意愿（"是否会截图发送"）

## 评估 Pipeline

```
代码提交 → 单元测试 → 集成测试 → 合规扫描 → 报告
   ↓          ↓          ↓          ↓
 通过率   覆盖率≥80%  黄金路径  0 违规
```

## 考点

- SWE-bench Verified 的改进方法
- 掌心人格局的测试分层
- 自动化评估 vs 人工评估的取舍
