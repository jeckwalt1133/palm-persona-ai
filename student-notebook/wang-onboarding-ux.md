# 新用户Onboarding引导流程

## 设计目标

用户从"这是什么东西"到"这个东西懂我"——3步完成首次人格报告，每步不超过1屏，不超过3句文案。

## 核心洞察

掌心人格局的onboarding有一个先天优势和一个先天劣势：
- **优势**：物理交互——拍照这个动作本身就有仪式感，比纯填表/问答有趣得多
- **劣势**：拍手掌是社交尴尬行为——用户可能在公共场合，或对"AI看手相"有防御心理

因此设计原则是：**降低尴尬感 + 加速到"啊哈"**。

---

## 第1步：拍照 → "这真的能看出什么吗？"

### 用户心理状态

| 维度 | 状态 |
|------|------|
| 情绪 | 好奇 + 轻微怀疑 + 社交自觉（"我在拍手掌，别人会不会觉得奇怪？"） |
| 认知负荷 | 中低 — 拍照这个动作本身很简单，但"该拍哪只手？""怎么摆？"需要一点引导 |
| 动力 | 中等 — 朋友分享来的（社交驱动力）> 自己发现的（好奇心驱动） |
| 防御 | 轻微 — "AI看手相"的刻板印象可能触发"这是迷信吧？"的质疑 |
| 期望 | 模糊 — 不太确定会看到什么，可能预期是类似星座测试的娱乐内容 |

### 文案（3句硬上限）

```
第1句（标题·降低尴尬）：伸出手掌，对着镜头——就像在看自己的手纹一样自然。

第2句（引导·消除不确定性）：不需要摆姿势，不需要调角度，手掌张开就行。

第3句（钩子·制造期待）：AI 会从你的掌心线条里读到一些你从没说过的事。
```

### 设计要点

- **第1句**：用"自然"一词直接对冲社交尴尬——你不是在做奇怪的事
- **第2句**：消除操作焦虑——"要不要很标准？""拍不好会不会不准？"→ 不需要
- **第3句**：把预期从"AI看手相"纠正为"读到没说出口的事"——从娱乐→自我认知

### 视觉配合
- 半透明手掌轮廓引导线（不是箭头，不是文字说明框）
- 一句话提示在镜头下方，字号不要太大（太大=好像在吼你）
- 相机快门按钮做成脉动效果——暗示"拍下来，AI就开始工作了"

## 第2步：等待 → "它在干嘛？准不准啊？"

### 用户心理状态

| 维度 | 状态 |
|------|------|
| 情绪 | 期待 + 轻度焦虑 + 自我怀疑（"等下会不会出来一堆瞎编的东西？"） |
| 认知负荷 | 低 — 只需要等，不需要操作 |
| 动力 | 升高 — 已经拍了，沉没成本让用户倾向于等到结果 |
| 防御 | 升高 — 等待期是用户最容易产生"这不会是骗人的吧"的时刻 |
| 期望 | 正在形成 — 脑子里已经在预演"等下AI会说我是什么样的人" |

**关键洞察**：等待是最危险的环节。信息真空会放大怀疑。必须用**具体进展**填满这个等待——不是进度百分比，是"AI正在发现什么"的预演。

### 文案（3句硬上限）

```
第1句（标题·承认等待）：正在读取你的掌心——这些线条每个人都不一样。

第2句（进展·制造预览）：已经看到了几条关键纹路，AI 正在把它们拼成你的故事。

第3句（钩子·提升预期）：快了——你第一次看到的结果，往往是最准的那一次。
```

### 设计要点

- **第1句**：不说"分析中请稍后"——说"每个人都不一样"，强化唯一性
- **第2句**：不说"特征提取50%"——说"正在把它们拼成你的故事"，把算法过程翻译成人类叙事
- **第3句**：最重要的锚定——"第一次最准"是为了防止用户测完说"我再测一次看看是不是随机生成的"。同时制造对第一次结果的珍惜感

### 等待动画
- 不是转圈不是进度条——是掌心纹路从模糊到清晰的渐变
- 5个步骤标签（沿用已实现的AnalyzeProgress.tsx文案）：
  - "看到你的掌纹了" → "这几条线有点意思" → "拼图快完成了" → "找到和你最像的那一类人" → "你的专属解读准备好了"
- 时间控制在8-12秒（太短=感觉没认真分析，太长=失去耐心）

## 第3步：查看 → "这说的真的是我吗？...等等，好像还真是。"

### 用户心理状态（啊哈时刻触发条件）

| 维度 | 状态 |
|------|------|
| 情绪 | 惊讶 + 被理解的温暖 + "我要发给XX看"的冲动 |
| 认知负荷 | 中 — 大量个人信息涌入，需要时间消化 |
| 动力 | 峰值 — 刚获得社交货币，分享冲动最强 |
| 防御 | 最低 — 被"说中了"之后，对AI的信任瞬间建立 |
| 期望 | 被超越 — 原以为类似星座，实际发现比星座具体得多 |

### "啊哈时刻"的定义

啊哈时刻不是"AI说对了我的性格"——那是预期内的。

啊哈时刻是：**AI说出了一件用户自己隐约知道、但从没对任何人说过的、关于自己的事。** 具体到能让用户心里咯噔一下的那种。

举例而非抽象：
- ❌ "你很敏感" → 这是任何星座都能说的话
- ✅ "你看起来很好说话，但心里有张严格的评分表" → 具体、伪装的社会面具被点破

### 首次报告"啊哈时刻"设计

首次报告的展示顺序不是按卡片类型排列——是按**情感打击力**排列：

```
打开报告后用户看到的顺序：

① 人格标签（大字，金色）
   "星辰梦想家" —— 一个让你想截图的标签
   → 反应: "哈，这名字有点意思"

② 核心真相（一句话，直接打脸）
   "你掌心清晰的纹路，藏着一个内心沉静的人——但骨子里，你是情绪细腻的。你比你表现出来的更复杂。"
   → 反应: "...等等，这说的好像真是我"

③ 视觉锚点（AI真的分析了你的手掌）
   "你的手掌偏宽，线条清晰，比85%的人多了2条线"
   → 反应: "它真的看了我的手掌？不是随便说的？"

④ 第1条洞察（最尖锐的那条）
   "你对美的感知力超乎常人——但你把它浪费在了'想'上，而不是'做'上。"
   → 反应: "靠。这是真的。"
   → ⚡ 这是啊哈时刻的核心触发器

⑤ 关键词标签（3个）
   "浪漫 · 直觉 · 创造力"
   → 反应: "我要发给我闺蜜/对象看"
   → 分享冲动触发
```

### 文案（3句硬上限 — 啊哈时刻后的承接）

```
第1句（确认·你被看见了）：AI 读到的这几行字，可能比你跟朋友说了一晚上的话还接近真实的你。

第2句（升级·这不是终点）：但这只是你人格的一个切面——下次再测，可能会看到不一样的自己。

第3句（行动·分享是自然的下一步）：如果你想到了某个人——发给ta。有些话AI替你说出来了。
```

### 设计要点

- **第1句**：不是"分析完成"，是"比聊天更接近真实的你"——确立AI的价值
- **第2句**：为留存埋钩子——"下次可能不一样"激励再测，也为人格变化提醒（retention copy #6）铺路
- **第3句**：把分享动作包装成"想到了某个人"而不是"邀请好友"——降低社交压力

## 啊哈时刻的强度分级

不是所有用户都会在同一瞬间"啊哈"。按人格类型和文案匹配度，啊哈时刻有三种强度：

| 强度 | 触发条件 | 用户反应 | 概率 |
|------|---------|---------|------|
| ⚡⚡⚡ 强啊哈 | 核心真相精确命中用户隐藏特质 | 截屏 → 分享 → "这AI怎么知道的？！" | 40% |
| ⚡⚡ 中啊哈 | 洞察部分命中，用户觉得"有道理" | 多翻几页 → 可能分享 | 45% |
| ⚡ 弱啊哈 | 文案偏通用，用户觉得"还行" | 划完关掉 → 可能不会回来 | 15% |

**提高强啊哈概率的措施**：
1. 视觉锚点放在核心真相前——先证明"AI真的分析了你的手掌"，再给真相
2. 核心真相用"温和版"还是"犀利版"取决于用户是否首次——首次默认温和，避免防御
3. 关键词必须精准到人格类型——"浪漫"配星辰梦想家，"可靠"配沉默守护者，不能互换
