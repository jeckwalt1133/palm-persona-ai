## 一、仪表盘设计原则

### 1.1 核心理念

> 不看虚荣指标，只看可操作指标。

- **虚荣指标**（不看）：累计用户数、页面浏览量（PV）、下载量
- **可操作指标**（看）：DAU、分享率、留存率、转化漏斗每一步的流失率

### 1.2 四层指标体系

```
L4 商业层  ─  广告收入 / 潜在订阅转化
    ↑
L3 转化层  ─  报告完成率 / 匹配创建率 / 分享率
    ↑
L2 活跃层  ─  DAU / 会话时长 / 功能使用频次
    ↑
L1 获客层  ─  新增用户 / 来源渠道 / 获客成本
```

## 二、核心指标定义 (16项)

### L1 获客层 (4项)

| # | 指标 | 英文 | 定义 | 计算方式 | 目标值(首月) |
|---|------|------|------|----------|-------------|
| 1 | **日活跃用户** | DAU | 每日至少打开一次小程序的独立设备数 | 每日0点重置的设备去重计数 | W1≥50 / W4≥150 |
| 2 | **日新增用户** | New Users | 每日首次打开小程序的独立设备数 | 设备ID首次出现在系统中的数量 | W1≥60/周 / 首月≥360 |
| 3 | **来源渠道分布** | Source Mix | 新用户各来源渠道占比 | 搜索/分享/公众号/朋友圈/抖音/其他 分组计数 | 自然搜索≥30% |
| 4 | **获客K因子** | K-Factor | 每用户平均带来的新用户数 | 邀请成功的新用户数 / 发起邀请的用户数 | W1≥1.2 / W4≥1.8 |

### L2 活跃层 (4项)

| # | 指标 | 英文 | 定义 | 计算方式 | 目标值(首月) |
|---|------|------|------|----------|-------------|
| 5 | **平均会话时长** | Avg Session | 用户每次打开小程序的平均停留秒数 | 总会话时长 / 总会话次数 | ≥180秒 |
| 6 | **人均日使用次数** | Daily Frequency | 活跃用户每天打开小程序的次数 | 日总打开次数 / DAU | ≥1.5次 |
| 7 | **功能渗透率** | Feature Penetration | 各核心功能的使用占比 | 使用过该功能的用户数 / DAU | 拍照≥80% / 标点≥60% / 问卷≥30% |
| 8 | **推送CTR** | Push CTR | 推送消息的点击率 | 推送点击次数 / 推送到达次数 | ≥12% |

### L3 转化层 (4项)

| # | 指标 | 英文 | 定义 | 计算方式 | 目标值(首月) |
|---|------|------|------|----------|-------------|
| 9 | **报告完成率** | Report Completion | 开始拍照到查看完整报告的比例 | 完成报告用户数 / 开始拍照用户数 | ≥65% |
| 10 | **匹配创建率** | Match Creation | 活跃用户中创建过匹配的比例 | 创建匹配用户数 / DAU | ≥15% |
| 11 | **分享率** | Share Rate | 报告页分享按钮点击率 | 点击分享次数 / 查看报告次数 | ≥25% |
| 12 | **内容转化率** | Content Conversion | 内容阅读→小程序打开的转化 | 小程序打开次数 / 内容阅读量 | ≥18% |

### L4 商业层 (4项)

| # | 指标 | 英文 | 定义 | 计算方式 | 目标值(首月) |
|---|------|------|------|----------|-------------|
| 13 | **激励视频完成率** | Ad Completion | 用户完整看完激励视频的比例 | 完整观看次数 / 广告触发次数 | ≥60% |
| 14 | **D1留存率** | D1 Retention | 新增用户次日回访比例 | D1回访用户数 / D0新增用户数 | ≥45% |
| 15 | **D7留存率** | D7 Retention | 新增用户第7日回访比例 | D7回访用户数 / D0新增用户数 | ≥38% |
| 16 | **D30留存率** | D30 Retention | 新增用户第30日回访比例 | D30回访用户数 / D0新增用户数 | ≥25% |

## 三、转化漏斗定义

### 3.1 主漏斗：首页 → 报告

```
                  首页曝光      100%  (所有打开用户)
                    │
                    ▼
                  开始拍照        ?%  (点击拍照按钮)
                    │
                    ▼
                  标点完成        ?%  (完成关键点标记)
                    │
                    ▼
                  问卷完成        ?%  (完成可选问卷)
                    │
                    ▼
                  AI分析中        ?%  (等待分析结果)
                    │
                    ▼
                  报告查看        ?%  (完整查看报告)
                    │
            ┌───────┴───────┐
            ▼               ▼
        分享海报         创建匹配
       (分享率?%)      (匹配率?%)
```

### 3.2 漏斗各步埋点事件

| 步骤 | 埋点事件名 | 触发时机 | 记录字段 |
|------|-----------|----------|----------|
| 首页曝光 | `page_view_home` | 首页渲染完成 | device_id, source, timestamp |
| 开始拍照 | `action_start_camera` | 点击拍照按钮 | device_id, from_page, timestamp |
| 拍照完成 | `action_capture_done` | 照片确认提交 | device_id, image_quality_score, file_size, timestamp |
| 标点开始 | `action_marking_start` | 进入辅助标点页 | device_id, timestamp |
| 标点完成 | `action_marking_done` | 所有关键点标记完成 | device_id, points_count, duration_ms, timestamp |
| 问卷开始 | `action_quiz_start` | 进入轻量问卷页 | device_id, timestamp |
| 问卷完成 | `action_quiz_done` | 问卷提交 | device_id, answers_count, duration_ms, timestamp |
| AI分析请求 | `api_analyze_request` | 发送分析请求 | device_id, provider, timestamp |
| AI分析响应 | `api_analyze_response` | 收到分析结果 | device_id, provider, latency_ms, success, timestamp |
| 报告查看 | `page_view_report` | 报告页渲染完成 | device_id, report_id, duration_to_read, timestamp |
| 分享点击 | `action_share_click` | 点击分享按钮 | device_id, report_id, share_channel, timestamp |
| 分享完成 | `action_share_done` | 分享成功 | device_id, report_id, share_channel, timestamp |
| 匹配创建 | `action_match_create` | 发起匹配邀请 | device_id, report_id, invite_method, timestamp |
| 匹配完成 | `action_match_done` | 双方均完成匹配查看 | device_id, match_id, match_score, timestamp |

### 3.3 漏斗目标转化率 (首月基线)

| 漏斗步骤 | 目标转化率 | 预警阈值 |
|----------|-----------|----------|
| 首页曝光 → 开始拍照 | ≥70% | <50% |
| 开始拍照 → 拍照完成 | ≥85% | <65% (图像质量拒绝率过高) |
| 拍照完成 → 标点完成 | ≥70% | <50% (标点体验问题) |
| 标点完成 → AI分析完成 | ≥95% | <85% (API故障) |
| AI分析完成 → 报告查看 | ≥90% | <75% |
| 报告查看 → 分享 | ≥25% | <15% |
| 报告查看 → 创建匹配 | ≥15% | <8% |

## 四、埋点技术方案

### 4.1 埋点架构

```
小程序端                     服务端
┌─────────────┐           ┌──────────────┐
│ 前端埋点SDK  │──POST──→│ /api/events   │
│ (精简版)     │           │ (批量接收)     │
└─────────────┘           └──────┬───────┘
                                 │
                          ┌──────▼───────┐
                          │ 事件流处理    │
                          │ (分类/聚合)   │
                          └──────┬───────┘
                                 │
                    ┌────────────┼────────────┐
                    ▼            ▼            ▼
              ┌─────────┐ ┌─────────┐ ┌─────────┐
              │ 实时看板 │ │ 日周报  │ │ 预警触发 │
              │ (内存)   │ │ (JSONL) │ │ (阈值)   │
              └─────────┘ └─────────┘ └─────────┘
```

### 4.2 埋点事件数据结构

```typescript
// 所有埋点事件统一结构
interface TrackEvent {
  event: string;              // 事件名 (见3.2)
  device_id: string;          // 设备匿名ID
  session_id: string;         // 会话ID (每次打开生成)
  timestamp: number;          // Unix毫秒时间戳
  source: string;             // 来源渠道: search/share/official_account/moments/douyin/other
  properties: Record<string, string | number | boolean>; // 事件属性
}

// 示例: 分享完成事件
{
  "event": "action_share_done",
  "device_id": "dev_abc123",
  "session_id": "sess_xyz789",
  "timestamp": 1715040000000,
  "source": "search",
  "properties": {
    "report_id": "rpt_001",
    "share_channel": "wechat_moments",
    "image_quality_score": 0.85
  }
}
```

### 4.3 埋点实施清单

| 埋点类型 | 实施方式 | 优先级 | 负责人 |
|----------|----------|--------|--------|
| 页面曝光 | 小程序页面 onShow 生命周期自动触发 | P0 | 前端 |
| 按钮点击 | 关键按钮绑定 track() 调用 | P0 | 前端 |
| API请求/响应 | 网络层拦截器统一埋点 | P0 | 后端 |
| 分享行为 | 微信/抖音分享回调中埋点 | P0 | 前端 |
| 广告行为 | 激励视频回调中埋点 | P1 | 前端 |
| 异常事件 | try/catch + 全局错误监听 | P0 | 前端+后端 |
| 来源追踪 | 小程序启动参数 scene + query 解析 | P0 | 前端 |

### 7.3 数据采集脚本 (伪代码)

```typescript
// 前端埋点函数 (精简版)
function track(event: string, properties: Record<string, any> = {}) {
  const payload: TrackEvent = {
    event,
    device_id: getDeviceId(),
    session_id: getSessionId(),
    timestamp: Date.now(),
    source: getSource(),
    properties
  };

  // 异步发送, 不阻塞主流程
  wx.request({
    url: `${API_BASE}/api/events`,
    method: 'POST',
    data: payload,
    // 失败不影响用户体验
    fail: () => console.debug('track failed:', event)
  });
}

// 关键位置调用示例
Page({
  onShow() {
    track('page_view_home');
  },
  onShareAppMessage() {
    track('action_share_done', {
      report_id: this.data.reportId,
      share_channel: 'wechat_friend'
    });
  }
});
```

## 八、首月复盘框架

### D+30 复盘检查清单

1. **核心指标达成**：16项指标 vs 目标值的偏差分析
2. **漏斗瓶颈识别**：转化率最低的步骤 → 优化优先级
3. **最佳内容识别**：CTR/分享率最高的3篇内容 → 提取模板
4. **渠道ROI排名**：搜索/分享/公众号/朋友圈/抖音 → 资源倾斜
5. **留存曲线分析**：D1/D7/D30留存率 → 判断产品粘性
6. **用户行为分群**：轻度(1-2次)/中度(3-5次)/重度(6+次)用户行为差异
7. **竞品对标**：与同赛道产品(测测/MBTI测试类)的指标对比
