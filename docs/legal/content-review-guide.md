# 内容审核指南

**最后更新：2026年5月7日**
**适用范围：掌心人格局小程序及 H5 版本**

本指南用于规范本产品所有用户生成内容的审核流程、违规判定标准和处置措施。

## 一、审核范围

### 1.1 需要审核的内容类型

| 内容类型 | 来源 | 风险等级 |
|---------|------|---------|
| 用户上传的手掌照片 | 拍照/相册选取 | 高 |
| AI 生成的分析报告文案 | 引擎自动生成 | 中 |
| 用户分享文案（自选） | 用户选择/自定义 | 中 |
| 用户反馈/意见建议 | 意见反馈入口 | 低 |
| 分享海报图片 | Canvas 自动生成 | 中 |

### 1.2 内容审核触发时机

- **实时审核**：手掌照片上传时（分析前）
- **生成审核**：AI 报告文案输出后（展示前）
- **发布审核**：分享内容发送前（社交传播前）

## 二、违规内容分类与判定标准

### 2.1 一类违规（零容忍，立即阻断+永久封禁）

| 违规类型 | 判定标准 | 示例 |
|---------|---------|------|
| 儿童性虐待材料 (CSAM) | 含未成年人裸露或性暗示的照片 | — |
| 极端暴力/恐怖主义 | 含血腥、暴力、恐怖组织标志的照片 | — |
| 裸露/色情 | 含性器官、性行为相关的照片 | 色情图片、露骨自拍 |
| 违法信息 | 含毒品、赌博、诈骗相关的照片或文字 | 赌博平台推广、诈骗链接 |

### 2.2 二类违规（阻断+警告）

| 违规类型 | 判定标准 | 示例 |
|---------|---------|------|
| 他人肖像 | 照片中含有可辨识的他人面部 | 自拍中含路人、明星照片 |
| 身份证/证件 | 照片中含有身份证、驾驶证等证件 | 拍手掌时拍到桌上证件 |
| 二维码/联系方式 | 照片中含有商业推广信息 | 含微信号二维码的照片 |
| 政治敏感 | 含政治人物、敏感旗帜、敏感文字 | — |

### 2.3 三类违规（替换文案+放行）

| 违规类型 | 判定标准 | 处置方式 |
|---------|---------|---------|
| 合规禁词 | AI 文案中出现"掌纹/手相/算命/注定"等 | 自动替换为合规表达 |
| 绝对化用语 | 文案中出现"100%/必然/一定/证明" | 弱化为"倾向于/大概率" |
| 敏感联想词 | "姻缘/正缘/宿命/克" | 替换为"相处适配度/互动模式" |

### 2.4 非违规正常内容

- 清晰的手掌照片（含手指、掌心纹路）
- 手部在自然光下的正常拍摄
- 含背景物体的手掌照片（不影响识别即可）
- AI 生成的合规人格分析文案

## 三、审核执行流程

### 3.1 图像审核流程

```
用户上传照片
  │
  ├─ 1. 格式校验：仅允许 JPEG/PNG/WebP，≤10MB
  │     └─ 不合格 → 提示"请上传 JPG/PNG 格式照片，大小不超过 10MB"
  │
  ├─ 2. 技术检测（服务端）
  │     ├─ 肤色面积占比检查（非手掌内容 > 60% → 拒绝）
  │     ├─ NSFW 内容检测（含色情/暴力 → 二类/一类违规）
  │     └─ 人脸检测（含可辨识人脸 → 二类违规）
  │
  ├─ 3. 违规判定
  │     ├─ 一类违规 → 拒绝 + 账号标记 + 人工复核
  │     ├─ 二类违规 → 拒绝 + 提示"请仅上传手掌照片，不要包含面部或其他个人信息"
  │     └─ 通过 → 进入分析引擎
  │
  └─ 4. 记录日志（不含原始照片）
```

### 3.2 AI 文案审核流程

```
AI 生成分析报告
  │
  ├─ 1. ContentSafety.filterNarrative() 扫描全文
  │     ├─ 匹配禁词库（50+ 词汇）→ 自动替换
  │     └─ 匹配绝对化句式 → 弱化改写
  │
  ├─ 2. 句式合规检查
  │     ├─ "你的XX证明" → "从XX特征看，你倾向于"
  │     └─ "你命中注定" → 阻断整个段落，重新生成
  │
  └─ 3. 最终人工检查输出（仅非法情况下触发）
        └─ 任何未匹配的异常 → 回退为安全兜底文案
```

### 3.3 审核时效要求

| 环节 | 时限 | 异常处理 |
|------|------|---------|
| 图片格式校验 | ≤100ms | 即时反馈 |
| NSFW/人脸检测 | ≤500ms | 超时 → 拒绝本次分析 |
| AI 文案过滤 | ≤200ms | 超时 → 展示通用安全文案 |
| 人工复核 | 24h 内 | 标记待复核队列 |

## 四、违规处置措施

### 4.1 处置分级

| 级别 | 措施 | 适用情形 |
|------|------|---------|
| L1 阻断 | 单次分析请求被拒绝，提示原因 | 二类违规（首犯） |
| L2 限制 | 当日分析次数清零，次日恢复 | 二类违规（再犯） |
| L3 封禁 | 永久禁止使用本产品 | 一类违规 / 累计 3 次二类违规 |

### 4.2 申诉机制

用户可通过"意见反馈"提交申诉：
1. 说明情况（附相关截图）
2. 我们将在 **3 个工作日内**复核
3. 复核结果通过产品内通知告知
4. 一类违规的人工复核为必经流程，不会仅凭自动检测封禁

## 五、审核人员操作规范

### 5.1 保密要求

- 审核人员不得截取、保存、传播任何用户上传内容
- 审核日志不记录原始照片
- 违反保密规定的人员将被立即移除审核权限

### 5.2 判断原则

- **疑罪从无**：无法确定的，先放行再标记观察
- **上下文判断**：结合用户行为模式，不孤立看单张照片
- **快速响应**：一类违规 1 小时内处理，二类 4 小时内

## 六、持续改进

### 6.1 审核标准更新

- 每周审查禁词库，根据新出现的违规模式更新
- 每月评估审核准确率（误拦率+漏拦率）
- 每季度对照最新法规和平台政策更新本指南

### 6.2 指标监控

| 指标 | 目标值 | 监控频率 |
|------|--------|---------|
| 误拦率（正常内容被拒） | < 3% | 周 |
| 漏拦率（违规内容通过） | < 0.1% | 周 |
| 审核响应时间 P99 | < 1s | 日 |
| 用户申诉率 | < 1% | 月 |
