# Review Report — 掌心人格局报告审查 Skill

## 触发条件

用户输入 `/review-report [report_id]`

- `report_id`: 报告ID（可选，缺省时审查当前对话中提到的报告）

## 核心审查维度

审查结果按严重程度分三级：

| 级别 | 标记 | 含义 |
|------|------|------|
| 🔴 阻断级 | MUST FIX | 违反合规红线，不改不可上线 |
| 🟡 严重级 | SHOULD FIX | 影响用户体验或共鸣感，建议修改 |
| 🔵 建议级 | NICE TO HAVE | 可优化但不强制 |

---

## 审查流程

### Phase 1: 获取报告数据

通过 MCP Server 获取报告：

```bash
# 方式1：通过 MCP 协议读取资源（推荐）
MCP Resource: palm://reports/{report_id}

# 方式2：直接调用 HTTP API
curl -s http://localhost:3001/api/reports/{report_id}
```

### Phase 2: 合规检查（🔴 阻断级）

逐项检查以下内容：

#### 2.1 禁用词扫描

检查报告文案（summary / insights / coreTruth / weeklyAdvice / scores.description / quote）是否包含以下禁用词：

```
命理类：掌纹、手相、看手相、算命、占卜
命运类：命运注定、天注定、宿命、改命、改运、开运
关系类：正缘、姻缘测算、旺夫、旺妻、克夫、克妻、天生一对
绝对化：100%准确、比算命更准、必然、一定会、暴富
灾祸类：寿命预测、疾病预测、灾祸预测、财富暴富预测
```

**通过标准**：零命中。

#### 2.2 替代表达使用

检查是否使用绝对化语气。以下表达不合格：

- ❌ "你一定是个..." → ✅ "你倾向于..."
- ❌ "这注定..." → ✅ "这大概率..."
- ❌ "这就是命运" → ✅ "趣味语境下，这反映了一种..."
- ❌ "必然导致" → ✅ "更容易带来"

**通过标准**：不得出现任何绝对化表达。

#### 2.3 免责声明

确认报告相关内容包含免责声明：
> "本产品为 AI 手掌图像趣味分析工具，结果仅供娱乐和自我探索，不构成任何专业建议。"

**通过标准**：报告中或报告所在页面必须包含。

### Phase 3: 质量检查（🟡 严重级）

#### 3.1 视觉锚点引用

每段分析必须引用具体手掌特征作为依据：

- ❌ "你是一个感性的人"（无依据）
- ✅ "你的感情线弧度较大（曲率0.72），倾向于感性地处理关系问题"

检查项：
- [ ] 报告是否有 `visualAnchors` 字段
- [ ] 分数描述中是否引用了具体特征（至少 3/5 的维度有）
- [ ] 核心真相和摘要是否体现了特征关联

**通过标准**：至少 60% 的分析段落包含具体特征引用。

#### 3.2 文案共鸣度

检查文案是否满足"截图驱动"标准：
- [ ] 是否有"让人想发给某个人"的句子
- [ ] 是否温和刺痛而非隔靴搔痒
- [ ] 是否有空泛套话（"你是一个善良、敏感、有潜力的人" → ❌）
- [ ] 是否有具体的场景化描述（"你真正累的不是事情多，而是总在照顾别人的感受" → ✅）

#### 3.3 有效内容密度

- [ ] insights 是否 ≥ 3 条
- [ ] 每条 insight 是否 ≥ 10 字
- [ ] scores 是否覆盖全部 5 个维度
- [ ] summary 是否 ≥ 30 字
- [ ] weeklyAdvice 是否具体可执行，而非"好好生活"式空泛

### Phase 4: 一致性检查（🔵 建议级）

#### 4.1 分数-描述匹配

分数是否与描述的语气一致：
- `score ≥ 80` 的描述应为积极正面
- `score 50-79` 的描述应中性或有建设性
- `score < 50` 的描述应温和且不负面

#### 4.2 特征-结论一致性

基于手掌特征得出的结论是否有逻辑：
- 生命线深长 ↔ 有持久力（不能说她善变）
- 智慧线长 ↔ 思维深入（不能说她冲动）

若发现不一致，标记为可疑。

#### 4.3 关键词覆盖

报告中的 `keywords` 是否与 `scores` 和 `insights` 内容一致：
- keywords 中的每个词是否在报告中有所体现
- 是否有未在 keywords 中列出但实际出现的关键主题

## 输出格式

审查完成后，输出结构化报告：

```json
{
  "report_id": "demo-001",
  "overall": "PASS / CONDITIONAL_PASS / FAIL",
  "checks": {
    "compliance": {
      "status": "PASS",
      "violations": [],
      "details": "合规检查通过，零禁用词命中"
    },
    "quality": {
      "status": "PASS",
      "score": 85,
      "items": [
        {"check": "视觉锚点引用", "status": "PASS", "note": "5/5 维度引用了特征"},
        {"check": "文案共鸣度", "status": "SHOULD_FIX", "note": "insights 第2条略显通用"}
      ]
    },
    "consistency": {
      "status": "PASS",
      "issues": [],
      "details": "分数-描述-关键词三方一致"
    }
  },
  "recommendations": [
    "🔵 建议级: insights[2] 可更具体"
  ]
}
```

## Definition of Done

- [ ] 全部 4 个 Phase 执行完毕
- [ ] 每个违规项标注了位置（哪段文案）
- [ ] 给出了可操作修改建议（不只是"不合格"）
- [ ] 输出结构化审查报告
